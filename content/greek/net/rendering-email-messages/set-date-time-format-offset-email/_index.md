# Ορισμός μορφής ημερομηνίας/ώρας και μετατόπισης ζώνης ώρας (Email)


## Εισαγωγή
Το GroupDocs.Viewer για .NET είναι ένα ισχυρό εργαλείο που επιτρέπει στους προγραμματιστές να ενσωματώνουν απρόσκοπτα δυνατότητες προβολής εγγράφων στις εφαρμογές .NET τους. Με το GroupDocs.Viewer, μπορείτε να προβάλλετε ένα ευρύ φάσμα μορφών εγγράφων, όπως PDF, έγγραφα του Microsoft Office, εικόνες και άλλα, απευθείας μέσα στην εφαρμογή σας, χωρίς να χρειάζεστε εξωτερικά πρόσθετα ή προγράμματα προβολής. Σε αυτό το ολοκληρωμένο σεμινάριο, θα σας καθοδηγήσουμε στη διαδικασία ρύθμισης του GroupDocs.Viewer για .NET, θα εξερευνήσουμε τις δυνατότητές του και θα σας δείξουμε πώς να το χρησιμοποιήσετε αποτελεσματικά για να βελτιώσετε τις δυνατότητες προβολής εγγράφων της εφαρμογής σας.
## Προαπαιτούμενα
Πριν ξεκινήσετε αυτό το σεμινάριο, βεβαιωθείτε ότι έχετε ρυθμίσει τις ακόλουθες προϋποθέσεις:
1. Visual Studio: Βεβαιωθείτε ότι έχετε εγκαταστήσει το Visual Studio στο σύστημά σας. Το GroupDocs.Viewer για .NET είναι πλήρως συμβατό με το Visual Studio, επιτρέποντας την απρόσκοπτη ενσωμάτωση στα έργα .NET σας.
2. GroupDocs.Viewer για .NET: Κατεβάστε και εγκαταστήστε το GroupDocs.Viewer για .NET από το [σύνδεσμος λήψης](https://releases.groupdocs.com/viewer/net/)Ακολουθήστε τις οδηγίες εγκατάστασης που παρέχονται για να ρυθμίσετε τη βιβλιοθήκη στο περιβάλλον ανάπτυξής σας.
3. .NET Framework: Βεβαιωθείτε ότι έχετε εγκαταστήσει την κατάλληλη έκδοση του .NET Framework. Το GroupDocs.Viewer για .NET υποστηρίζει διάφορες εκδόσεις του .NET Framework, συμπεριλαμβανομένων των .NET Core και .NET Standard.

## Εισαγωγή χώρων ονομάτων
Για να χρησιμοποιήσετε αποτελεσματικά το GroupDocs.Viewer για .NET, πρέπει να εισαγάγετε τους απαραίτητους χώρους ονομάτων στο έργο σας. Ακολουθήστε τα παρακάτω βήματα για να εισαγάγετε τους απαιτούμενους χώρους ονομάτων:

```csharp
using System;
using System.IO;
using GroupDocs.Viewer.Options;
```


Ας αναλύσουμε το παράδειγμα που παρέχεται σε πολλά βήματα για να κατανοήσουμε κάθε στοιχείο και τη λειτουργικότητά του.
## Βήμα 1: Ορισμός καταλόγου εξόδου και διαδρομής αρχείου
```csharp
string outputDirectory = "Your Document Directory";
string filePath = Path.Combine(outputDirectory, "output.html");
```
Σε αυτό το βήμα, ορίζουμε τον κατάλογο εξόδου όπου θα αποθηκευτεί το αποδοσμένο έγγραφο και καθορίζουμε τη διαδρομή αρχείου για το αρχείο HTML εξόδου.
## Βήμα 2: Δημιουργία αντικειμένου προβολής
```csharp
using (Viewer viewer = new Viewer(TestFiles.SAMPLE_EML))
```
Εδώ, δημιουργούμε μια νέα παρουσία του `Viewer` κλάση, περνώντας τη διαδρομή του εγγράφου που θα προβληθεί (σε αυτήν την περίπτωση, ένα δείγμα αρχείου EML) ως παράμετρο.
## Βήμα 3: Ορισμός επιλογών προβολής HTML
```csharp
HtmlViewOptions options = HtmlViewOptions.ForEmbeddedResources(filePath);
```
Σε αυτό το βήμα, ρυθμίζουμε τις επιλογές προβολής HTML για την απόδοση του εγγράφου, καθορίζοντας τη διαδρομή αρχείου εξόδου για το έγγραφο HTML που έχει αποδοθεί.
## Βήμα 4: Ορισμός μορφής ημερομηνίας/ώρας και μετατόπισης ζώνης ώρας
```csharp
options.EmailOptions.DateTimeFormat = "MM d yyyy HH:mm tt zzz";
options.EmailOptions.TimeZoneOffset = new TimeSpan(1, 0, 0);
```
Εδώ, προσαρμόζουμε τη μορφή ημερομηνίας και ώρας για τα μηνύματα email και ορίζουμε την απόκλιση ζώνης ώρας σύμφωνα με την επιθυμητή ζώνη ώρας.
## Βήμα 5: Απόδοση εγγράφου
```csharp
viewer.View(options);
```
Τέλος, ονομάζουμε το `View` μέθοδος του `Viewer` αντικείμενο, μεταβιβάζοντας τις διαμορφωμένες επιλογές προβολής HTML για την απόδοση του εγγράφου σε μορφή HTML.
## Βήμα 6: Εμφάνιση καταλόγου εξόδου
```csharp
Console.WriteLine($"\nSource document rendered successfully.\nCheck output in {outputDirectory}.");
```
Αυτό το βήμα εμφανίζει απλώς ένα μήνυμα που υποδεικνύει την επιτυχή απόδοση του εγγράφου και παρέχει τη διαδρομή προς τον κατάλογο εξόδου όπου βρίσκεται το έγγραφο HTML που έχει αποδοθεί.
