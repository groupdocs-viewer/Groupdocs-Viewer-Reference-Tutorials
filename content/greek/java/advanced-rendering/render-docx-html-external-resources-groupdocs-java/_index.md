## Οδηγός Υλοποίησης

Ας διασπάσουμε την υλοποίηση σε σαφείς ενότητες:

### Απόδοση Εγγράφου με Εξωτερικούς Πόρους
Αυτή η δυνατότητα σας επιτρέπει να μετατρέψετε ένα αρχείο DOCX σε μορφή HTML διατηρώντας όλους τους εξωτερικούς πόρους όπως εικόνες ξεχωριστά αλλά προσβάσιμα.

#### Διαδικασία Βήμα‑Βήμα
1. **Ορισμός Καταλόγου Εξόδου και Μορφών Αρχείων**  
   Ορίστε διαδρομές για την αποθήκευση των αρχείων εξόδου, συμπεριλαμβανομένων των συμβάσεων ονοματοδοσίας για σελίδες και πόρους:

```java
String outputDirectory = "YOUR_OUTPUT_DIRECTORY/RenderToHtmlWithExternalResources";
String pageFilePathFormat = outputDirectory + "/page_{0}.html"; // Naming pattern for HTML pages
String resourceFilePathFormat = outputDirectory + "/page_{0}_{1}"; // Pattern for resources (e.g., images)
String resourceUrlFormat = outputDirectory + "/page_{0}_{1}"; // URL format in generated HTML
```

2. **Διαμόρφωση HtmlViewOptions**  
   Ενημερώστε το viewer να δημιουργεί εξωτερικούς πόρους χρησιμοποιώντας τις διαδρομές που ορίσατε:

```java
HtmlViewOptions viewOptions = HtmlViewOptions.forExternalResources(pageFilePathFormat, resourceFilePathFormat, resourceUrlFormat);
```

3. **Αρχικοποίηση και Απόδοση του Εγγράφου**  
   Χρησιμοποιήστε την κλάση `Viewer` για να επεξεργαστείτε το DOCX σύμφωνα με τις παραπάνω επιλογές:

```java
try (Viewer viewer = new Viewer(TestFiles.SAMPLE_DOCX)) {
    viewer.view(viewOptions); // Renders DOCX as HTML with external resources
}
```

#### Κύριες Επιλογές Διαμόρφωσης
- **`HtmlViewOptions.forExternalResources()`** – σας επιτρέπει να ελέγχετε πού γράφονται οι σελίδες HTML και τα περιουσιακά στοιχεία και πώς αναφέρονται στο παραγόμενο markup.  
- Τα placeholders (`{0}`, `{1}`) αντικαθίστανται κατά την εκτέλεση με αριθμούς σελίδας και αναγνωριστικά πόρων, εξασφαλίζοντας ότι κάθε αρχείο λαμβάνει μοναδικό όνομα.

### Συμβουλές Επίλυσης Προβλημάτων
- Επαληθεύστε ότι ο κατάλογος εξόδου υπάρχει και ότι η εφαρμογή έχει δικαιώματα εγγραφής.
