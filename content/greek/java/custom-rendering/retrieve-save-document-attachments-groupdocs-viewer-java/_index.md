## Γρήγορες Απαντήσεις

- **Είναι αυτή η προσέγγιση thread‑safe;** Δημιουργήστε ξεχωριστό αντικείμενο `Viewer` ανά νήμα ή συγχρονίστε την πρόσβαση.

## Τι είναι το java file output stream;
`java.io.FileOutputStream` είναι μια βασική κλάση της Java που γράφει ακατέργαστα byte σε αρχείο δυαδικό περι ή οποιαδήποτε μη‑κείμενα δεδομένα που εξάγονται από ένα έγγραφο.

## Γιατί να χρησιμοποιήσετε java file output stream με το GroupDocs.Viewer;
- **Άμεση διαχείριση δυαδικών δεδομένων** – Δεν απαιτούνται ενδιάμεσες μετατροπές· τα byte του συνημμένου γράφονται ακριβώς όπως εμφανίζονται στο αρχικό αρχείο.  
- **Απόδοση** – Η εγγραφή μέσω ροής ελαχιστοποιεί τη χρήση μνήμης, ειδικά για μεγάλα συνημμένα.

## Οδηγός Υλοποίησης

### Ανάκτηση Συνημμένων Χρησιμοποιώντας java file output stream
Ας αναλύσουμε πώς μπορείτε να ανακτήσετε συνημμένα χρησιμοποιώντας το GroupDocs.Viewer. Αυτή η δυνατότητα σας επιτρέπει να εξάγετε μεταδεδομένα για κάθε συνημμένο που υπάρχει στο έγγραφό σας.

#### Επισκόπηση
Η ανάκτηση συνημμένων περιλαμβάνει την πρόσβαση στη μέθοδο `getAttachments`, η οποία επιστρέφει μια λίστα αντικειμένων `Attachment` που περιέχουν λεπτομέρειες όπως το όνομα αρχείου και το μέγεθος.

#### Βήματα Υλοποίησης
1. **Δημιουργήστε ένα αντικείμενο Viewer**  
   Αρχικοποιήστε την κλάση `Viewer` με τη διαδρομή του εγγράφου σας:

   ```java
   import com.groupdocs.viewer.Viewer;
   import com.groupdocs.viewer.examples.TestFiles;
   import com.groupdocs.viewer.results.Attachment;

   public class FeatureRetrieveAttachments {
       public static void main(String[] args) {
           try (Viewer viewer = new Viewer(TestFiles.SAMPLE_MSG_WITH_ATTACHMENTS)) {
               // Proceed to retrieve attachments
           } catch (Exception e) {
               throw new RuntimeException(e);
           }
       }
   }
   ```

2. **Ανακτήστε τα Συνημμένα**  
   Χρησιμοποιήστε τη μέθοδο `getAttachments`:

   ```java
   List<Attachment> attachments = viewer.getAttachments();
   for (Attachment attachment : attachments) {
       System.out.println(attachment.getFileName());
   }
   ```

3. **Κατανόηση Παραμέτρων και Μεθόδων**  
   - `Viewer`: Δέχεται διαδρομή αρχείου ή ροή προς το έγγραφο.  
   - `getAttachments()`: Ανακτά μια λίστα συνημμένων αρχείων, παρέχοντας λεπτομέρειες όπως τα ονόματα.

### Αποθήκευση Συνημμένων Εγγράφου σε Κατάλογο
Τώρα που έχετε μάθει πώς να ανακτήσετε τα συνημμένα, ας τα αποθηκεύσουμε στον επιθυμητό κατάλογο χρησιμοποιώντας το API GroupDocs.Viewer και **java file output stream**.

#### Επισκόπηση
Αυτή η δυνατότητα περιλαμβάνει την αποθήκευση κάθε ανακτημένου αρχείου συνημμένου σε έναν καθορισμένο φάκελο εξόδου.

#### Βήματα Υλοποίησης
1. **Ορίστε τον Φάκελο Εξόδου**  
   Ορίστε μια διαδρομή `outputDirectory` όπου θα αποθηκευτούν τα αρχεία:

   ```java
   import java.nio.file.Path;
   import java.nio.file.Paths;

   Path outputDirectory = Paths.get("YOUR_OUTPUT_DIRECTORY");
   ```

2. **Αποθηκεύστε τα Συνημμένα**  
   Χρησιμοποιήστε βρόχο για να αποθηκεύσετε κάθε συνημμένο με τη μέθοδο `saveAttachment`:

   ```java
   try (Viewer viewer = new Viewer(TestFiles.SAMPLE_MSG_WITH_ATTACHMENTS)) {
       List<Attachment> attachments = viewer.getAttachments();
       
       for (Attachment attachment : attachments) {
           final Path file = outputDirectory.resolve(attachment.getFileName());
           try (FileOutputStream outputStream = new FileOutputStream(file.toFile())) {
               viewer.saveAttachment(attachment, outputStream);
           }
       }
   } catch (Exception e) {
       throw new RuntimeException(e);
   }
   ```

3. **Εξήγηση Παραμέτρων και Μεθόδων**  
   - `saveAttachment`: Δέχεται ένα αντικείμενο `Attachment` και ένα file output stream για την αποθήκευση του συνημμένου.  
   - `FileOutputStream`: Διαχειρίζεται τη γραφή δεδομένων σε αρχεία χρησιμοποιώντας τη λογική του **java file output stream**.
