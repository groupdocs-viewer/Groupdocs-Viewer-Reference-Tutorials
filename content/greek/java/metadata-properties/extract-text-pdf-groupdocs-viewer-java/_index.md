# Πώς να Εξάγετε Κείμενο PDF Χρησιμοποιώντας το GroupDocs.Viewer για Java

Η εξαγωγή κειμένου από PDF είναι μια βασική απαίτηση για πολλές εφαρμογές που βασίζονται σε δεδομένα. Σε αυτό το tutorial θα σας καθοδηγήσουμε **πώς να εξάγετε pdf** περιεχόμενο αποδοτικά με τη βιβλιοθήκη **GroupDocs Viewer Java**. Είτε χρειάζεστε την ευρετηρίαση εγγράφων, την εκτέλεση αναλύσεων ή τη μετεγκατάσταση παλαιών αρχείων, τα παρακάτω βήματα παρέχουν μια πλήρη, έτοιμη για παραγωγή λύση.

## Γρήγορες Απαντήσεις
- **Ποια βιβλιοθήκη είναι η καλύτερη για εξαγωγή κειμένου pdf;** GroupDocs.Viewer Java παρέχει ένα ισχυρό pdf text extraction api.  
- **Μπορώ να εξάγω κείμενο από PDF πολλαπλών σελίδων;** Ναι – ο viewer επαναλαμβάνει αυτόματα κάθε σελίδα και γραμμή.  
- **Χρειάζομαι άδεια για παραγωγή;** Απαιτείται εμπορική άδεια· διατίθεται δωρεάν δοκιμή για αξιολόγηση.  
- **Ποια έκδοση Java υποστηρίζεται;** JDK 1.8+ (λειτουργούν επίσης οι τελευταίες εκδόσεις LTS).  
- **Είναι το Maven ο μόνος τρόπος για να προσθέσετε την εξάρτηση;** Το Maven συνιστάται, αλλά μπορείτε επίσης να χρησιμοποιήσετε Gradle ή χειροκίνητη προσθήκη JAR.

## Τι είναι η Εξαγωγή Κειμένου PDF και γιατί να χρησιμοποιήσετε το GroupDocs Viewer;
Το **pdf text extraction api** διαβάζει το κειμενικό στρώμα ενός PDF χωρίς να αποδίδει το οπτικό περιεχόμενο. Αυτή η προσέγγιση είναι πολύ πιο γρήγορη από το OCR βασισμένο σε ραστερ και διατηρεί την αρχική δομή του εγγράφου. Το GroupDocs Viewer Java προσθέτει επιπλέον αξία χειρίζοντας σύνθετες διατάξεις, κρυπτογραφημένα αρχεία και έγγραφα πολλαπλών σελίδων έτοιμα για χρήση.

## Προαπαιτούμενα
- **Java Development Kit (JDK) 1.8+** εγκατεστημένο.  
- **Maven** για διαχείριση εξαρτήσεων (ή Gradle αν προτιμάτε).  
- Πρόσβαση σε άδεια **GroupDocs Viewer for Java** (δωρεάν δοκιμή ή αγορασμένη).  
- Βασικές γνώσεις Java – θα γράψετε μερικά μπλοκ `try‑with‑resources`.

## Οδηγός Υλοποίησης
Ακολουθεί ένας σύντομος, βήμα‑βήμα οδηγός για το πώς να εξάγετε κείμενο PDF με το GroupDocs Viewer Java.

### 1. Αρχικοποίηση του Αντικειμένου Viewer
```java
try (Viewer viewer = new Viewer("YOUR_DOCUMENT_DIRECTORY/SAMPLE_PDF")) {
    // Initialization complete, proceed to next steps.
}
```
Η παρουσία `Viewer` δείχνει στο PDF που θέλετε να επεξεργαστείτε. Η χρήση ενός μπλοκ *try‑with‑resources* εγγυάται ότι οι εγγενείς πόροι απελευθερώνονται αυτόματα.

### 2. Διαμόρφωση του `ViewInfoOptions` για Εξαγωγή Κειμένου
```java
ViewInfoOptions viewInfoOptions = ViewInfoOptions.forHtmlView();
viewInfoOptions.setExtractText(true);
```
Η ρύθμιση `setExtractText(true)` ενημερώνει το **pdf text extraction api** να συμπεριλάβει το ακατέργαστο κείμενο στις πληροφορίες προβολής.

### 3. Ανάκτηση Πληροφοριών Εγγράφου
```java
PdfViewInfo viewInfo = (PdfViewInfo) viewer.getViewInfo(viewInfoOptions);
```
`PdfViewInfo` σας δίνει πρόσβαση σε κάθε σελίδα, γραμμή και την κειμενική της τιμή.

### 4. Επανάληψη Μέσω Σελίδων και Γραμμών (Εξαγωγή Κειμένου PDF Πολλαπλών Σελίδων)
```java
for (Page page : viewInfo.getPages()) {
    for (Line line : page.getLines()) {
        System.out.println(line.getValue());
    }
}
```
Αυτός ο βρόχος εκτυπώνει κάθε γραμμή κειμένου, διαχειριζόμενος αυτόματα σενάρια **extract multi page pdf**. Μπορείτε να αντικαταστήσετε το `System.out.println` με κώδικα που γράφει σε αρχείο, βάση δεδομένων ή ευρετήριο αναζήτησης.

#### Συμβουλές Επίλυσης Προβλημάτων
- Ελέγξτε ξανά τη διαδρομή του αρχείου· λανθασμένη διαδρομή προκαλεί `FileNotFoundException`.  
- Βεβαιωθείτε ότι καλείται το `setExtractText(true)`· διαφορετικά επιστρέφονται μόνο οπτικά δεδομένα.  
- Για κρυπτογραφημένα PDF, περάστε τον κωδικό μέσω του υπερφορτωμένου κατασκευαστή `Viewer`.

Κατά την εργασία με μεγάλα αρχεία ή παρτίδες εργασιών:
