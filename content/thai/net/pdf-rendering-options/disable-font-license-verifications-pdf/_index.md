# ปิดใช้งานการตรวจสอบใบอนุญาตแบบอักษรใน PDF

## การแนะนำ
ในการพัฒนา .NET การจัดการและแก้ไขเอกสารถือเป็นส่วนสำคัญของแอปพลิเคชันต่างๆ มากมาย ไม่ว่าจะเป็นการดู PDF เอกสาร Word หรือไฟล์ประเภทอื่นๆ การมีเครื่องมือที่มีประสิทธิภาพในการจัดการงานเหล่านี้อย่างมีประสิทธิภาพถือเป็นสิ่งสำคัญ ซึ่งนี่คือจุดที่ GroupDocs.Viewer สำหรับ .NET เข้ามามีบทบาท ไลบรารีอันทรงพลังนี้ช่วยให้ผู้พัฒนาสามารถผสานรวมฟังก์ชันการดูเอกสารเข้ากับแอปพลิเคชัน .NET ได้อย่างราบรื่น

![ปิดใช้งานการตรวจสอบใบอนุญาตแบบอักษรใน PDF ด้วย GroupDocs.Viewer .NET](/viewer/pdf-rendering-options/disable-font-license-verifications-in-pdf.png)

## ข้อกำหนดเบื้องต้น
ก่อนที่จะเริ่มใช้ GroupDocs.Viewer สำหรับ .NET คุณต้องมีข้อกำหนดเบื้องต้นบางประการดังนี้:
### 1. ติดตั้ง Visual Studio
อันดับแรกสุด ตรวจสอบให้แน่ใจว่าคุณได้ติดตั้ง Visual Studio ไว้ในระบบของคุณแล้ว คุณสามารถดาวน์โหลดได้จากเว็บไซต์ของ Microsoft หากยังไม่ได้ดาวน์โหลด
### 2. ดาวน์โหลด GroupDocs.Viewer สำหรับ .NET
มุ่งหน้าไปที่ [ลิงค์ดาวน์โหลด](https://releases.groupdocs.com/viewer/net/) เพื่อรับ GroupDocs.Viewer เวอร์ชันล่าสุดสำหรับ .NET ปฏิบัติตามคำแนะนำในการติดตั้งที่ให้ไว้เพื่อตั้งค่าภายในสภาพแวดล้อมการพัฒนาของคุณ
### 3. การขอใบอนุญาตชั่วคราว
หากต้องการปลดล็อกศักยภาพทั้งหมดของ GroupDocs.Viewer สำหรับ .NET ในระหว่างการพัฒนาและการทดสอบ ขอแนะนำให้ขอรับใบอนุญาตชั่วคราว คุณสามารถขอได้จาก [ที่นี่](https://purchase-groupdocs.com/temporary-license/).

## นำเข้าเนมสเปซ
เมื่อคุณดำเนินการตามข้อกำหนดเบื้องต้นเสร็จเรียบร้อยแล้ว คุณก็พร้อมที่จะเริ่มใช้ GroupDocs.Viewer สำหรับ .NET ในโปรเจ็กต์ของคุณได้แล้ว เริ่มต้นด้วยการนำเข้าเนมสเปซที่จำเป็นลงในฐานโค้ดของคุณ
```csharp
using System;
using System.IO;
using GroupDocs.Viewer.Options;
```

ให้เราแบ่งตัวอย่างที่ให้ไว้ออกเป็นหลายขั้นตอนเพื่อความเข้าใจที่ชัดเจนยิ่งขึ้น:
## ขั้นตอนที่ 1: กำหนดไดเรกทอรีผลลัพธ์
เริ่มต้นด้วยการกำหนดไดเรกทอรีที่คุณต้องการจัดเก็บหน้าเอกสารที่แสดงผล
```csharp
string outputDirectory = "Your Document Directory";
```
## ขั้นตอนที่ 2: กำหนดรูปแบบเส้นทางไฟล์เพจ
กำหนดรูปแบบสำหรับเส้นทางไฟล์ของแต่ละหน้าของเอกสาร
```csharp
string pageFilePathFormat = Path.Combine(outputDirectory, "page_{0}.png");
```
## ขั้นตอนที่ 3: เริ่มต้น Viewer Object
สร้างอินสแตนซ์ของคลาส Viewer โดยส่งเส้นทางไปยังเอกสารที่คุณต้องการดู
```csharp
using (Viewer viewer = new Viewer(TestFiles.OXPS_EMBEDDED_FONT))
```
## ขั้นตอนที่ 4: กำหนดค่าตัวเลือกมุมมอง HTML
กำหนดตัวเลือกสำหรับการดูเอกสารเป็น HTML โดยระบุรูปแบบสำหรับทรัพยากรที่ฝังไว้ (เช่น รูปภาพ)
```csharp
HtmlViewOptions options = HtmlViewOptions.ForEmbeddedResources(pageFilePathFormat);
```
## ขั้นตอนที่ 5: ปิดใช้งานการตรวจสอบใบอนุญาตแบบอักษร
เปิดใช้งานตัวเลือกในการปิดใช้งานการตรวจสอบใบอนุญาตแบบอักษรเพื่อให้แน่ใจว่าจะแสดงผลได้อย่างราบรื่น
```csharp
options.PdfOptions.DisableFontLicenseVerifications = true;
```
## ขั้นตอนที่ 6: ดูเอกสาร
เรียกใช้วิธีการมุมมองของวัตถุ Viewer โดยส่งตัวเลือกที่กำหนดค่าไว้
```csharp
viewer.View(options);
```
## ขั้นตอนที่ 7: แสดงไดเรกทอรีผลลัพธ์
แจ้งให้ผู้ใช้ทราบเกี่ยวกับตำแหน่งที่เก็บหน้าเอกสารที่แสดงผล
```csharp
Console.WriteLine($"\nSource document rendered successfully.\nCheck output in {outputDirectory}.");
```

## บทสรุป
GroupDocs.Viewer สำหรับ .NET นำเสนอโซลูชันที่ครอบคลุมสำหรับนักพัฒนาในการผสานรวมความสามารถในการดูเอกสารเข้ากับแอปพลิเคชัน .NET ได้อย่างง่ายดาย เพียงทำตามขั้นตอนที่ระบุไว้ในบทช่วยสอนนี้ คุณจะสามารถใช้ไลบรารีอันทรงพลังนี้เพื่อปรับปรุงเวิร์กโฟลว์การจัดการเอกสารของคุณได้อย่างมีประสิทธิภาพ
## คำถามที่พบบ่อย
### GroupDocs.Viewer สำหรับ .NET สามารถจัดการกับรูปแบบเอกสารหลายรูปแบบได้หรือไม่
ใช่ GroupDocs.Viewer รองรับรูปแบบเอกสารต่างๆ มากมาย รวมถึง PDF, Microsoft Word, Excel, PowerPoint และอื่นๆ อีกมากมาย
### GroupDocs.Viewer สำหรับ .NET เหมาะกับแอพพลิเคชันเว็บหรือไม่
แน่นอนว่า GroupDocs.Viewer สามารถผสานรวมเข้ากับแอพพลิเคชันเดสก์ท็อปและเว็บที่พัฒนาโดยใช้เทคโนโลยี .NET ได้อย่างราบรื่น
### GroupDocs.Viewer ต้องมีการอ้างอิงเพิ่มเติมใด ๆ หรือไม่
ไม่ GroupDocs.Viewer สำหรับ .NET มีการอ้างอิงขั้นต่ำและสามารถรวมเข้ากับโปรเจ็กต์ที่มีอยู่ของคุณได้อย่างง่ายดาย
### ฉันสามารถปรับแต่งลักษณะที่ปรากฏของเอกสารที่แสดงผลได้หรือไม่
ใช่ GroupDocs.Viewer มีตัวเลือกต่าง ๆ ในการปรับแต่งลักษณะที่ปรากฏและพฤติกรรมของเอกสารที่แสดงผลเพื่อให้เหมาะกับความต้องการเฉพาะของคุณ
### มีการสนับสนุนด้านเทคนิคสำหรับ GroupDocs.Viewer สำหรับ .NET หรือไม่
ใช่ คุณสามารถขอความช่วยเหลือและคำแนะนำจากทีมสนับสนุนเฉพาะทางได้ผ่านทาง [ฟอรั่ม](https://forum-groupdocs.com/c/viewer/9).
