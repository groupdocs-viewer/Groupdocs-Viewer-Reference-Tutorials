# โหลดเอกสารจากดิสก์ภายในเครื่อง

## การแนะนำ
ในยุคดิจิทัลทุกวันนี้ การเรนเดอร์เอกสารอย่างมีประสิทธิภาพถือเป็นสิ่งสำคัญสำหรับแอปพลิเคชันต่างๆ Groupdocs.Viewer สำหรับ .NET นำเสนอโซลูชันอันทรงพลังสำหรับการเรนเดอร์เอกสารโดยตรงจากดิสก์ภายในเครื่องของคุณ ในบทช่วยสอนนี้ เราจะแนะนำคุณตลอดกระบวนการโหลดเอกสารจากดิสก์ภายในเครื่องของคุณโดยใช้ Groupdocs.Viewer สำหรับ .NET ไม่ว่าคุณจะเป็นนักพัฒนาที่มีประสบการณ์หรือเพิ่งเริ่มต้น คู่มือทีละขั้นตอนนี้จะช่วยให้คุณผสานการเรนเดอร์เอกสารเข้ากับแอปพลิเคชัน .NET ของคุณได้อย่างราบรื่น

![โหลดเอกสารจากดิสก์ภายในเครื่องด้วย GroupDocs.Viewer .NET](/viewer/loading-documents/load-documents-from-local-disk.png)

## ข้อกำหนดเบื้องต้น
ก่อนจะเริ่มบทช่วยสอนนี้ ให้แน่ใจว่าคุณมีข้อกำหนดเบื้องต้นดังต่อไปนี้:
1. Groupdocs.Viewer สำหรับ .NET: ดาวน์โหลดและติดตั้งเวอร์ชันล่าสุดจาก [ที่นี่](https://releases-groupdocs.com/viewer/net/).
2. สภาพแวดล้อมการพัฒนา .NET: ตรวจสอบให้แน่ใจว่าคุณมีการตั้งค่าสภาพแวดล้อมการพัฒนา .NET ที่ทำงานอยู่บนระบบของคุณแล้ว
3. เอกสารภายในเครื่อง: จัดเก็บเอกสารที่คุณต้องการเรนเดอร์ไว้ในเครื่องบนดิสก์ของคุณ

## นำเข้าเนมสเปซ
ก่อนอื่นให้เรานำเข้าเนมสเปซที่จำเป็นเพื่อเข้าถึงฟังก์ชันการทำงานของ Groupdocs.Viewer สำหรับ .NET
```csharp
using System;
using System.IO;
using GroupDocs.Viewer.Options;
```
## ขั้นตอนที่ 1: โหลดเอกสารจากดิสก์ภายในเครื่อง
เริ่มต้นด้วยการตั้งค่าไดเร็กทอรีเอาต์พุตที่จะบันทึกหน้า HTML ที่จะแสดงผล
```csharp
string outputDirectory = "Your Document Directory";
string pageFilePathFormat = Path.Combine(outputDirectory, "page_{0}.html");
```
## ขั้นตอนที่ 2: เริ่มต้นโปรแกรมดูและเรนเดอร์เอกสาร
สร้างการเริ่มต้นวัตถุ Viewer ด้วยเส้นทางของเอกสาร และเรนเดอร์โดยใช้ตัวเลือกมุมมอง HTML
```csharp
using (Viewer viewer = new Viewer("Path_to_Your_Document"))
{
    HtmlViewOptions options = HtmlViewOptions.ForEmbeddedResources(pageFilePathFormat);
    viewer.View(options);
}
```
## ขั้นตอนที่ 3: แสดงผลลัพธ์
เมื่อการเรนเดอร์เสร็จสิ้น ให้แสดงข้อความแจ้งการเรนเดอร์เอกสารต้นฉบับสำเร็จและตำแหน่งของไฟล์เอาต์พุต
```csharp
Console.WriteLine($"\nSource document rendered successfully.\nCheck output in {outputDirectory}.");
```
