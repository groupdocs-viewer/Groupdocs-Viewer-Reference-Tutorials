# เรนเดอร์ด้วยแบบอักษรที่กำหนดเอง

## การแนะนำ
GroupDocs.Viewer เป็นโซลูชันอันทรงพลังสำหรับการแสดงผลเอกสารในรูปแบบต่างๆ ในด้านการพัฒนา .NET โดย GroupDocs.Viewer มีความสามารถมากมายในการให้การแสดงผลเอกสารด้วยแบบอักษรที่กำหนดเอง ซึ่งช่วยเพิ่มความเป็นส่วนตัวและความยืดหยุ่นให้กับแอปพลิเคชันของคุณ
## ข้อกำหนดเบื้องต้น
ก่อนจะดำเนินการเรนเดอร์เอกสารด้วยแบบอักษรที่กำหนดเองโดยใช้ GroupDocs.Viewer สำหรับ .NET ให้แน่ใจว่าคุณมีข้อกำหนดเบื้องต้นต่อไปนี้:
### 1. ติดตั้ง GroupDocs.Viewer สำหรับ .NET
หากต้องการใช้ GroupDocs.Viewer สำหรับ .NET คุณจะต้องติดตั้งไว้ในสภาพแวดล้อมการพัฒนาของคุณก่อน คุณสามารถดาวน์โหลดแพ็คเกจที่จำเป็นได้จากลิงก์ที่ให้มา:
[ดาวน์โหลด GroupDocs.Viewer สำหรับ .NET](https://releases.groupdocs.com/viewer/net/)
### 2. รับแบบอักษร
เตรียมแบบอักษรที่กำหนดเองที่คุณต้องการใช้ในการเรนเดอร์เอกสาร ตรวจสอบให้แน่ใจว่าสามารถเข้าถึงแบบอักษรเหล่านี้ได้ภายในสภาพแวดล้อมแอปพลิเคชันของคุณ
### 3. ตั้งค่าสภาพแวดล้อมการพัฒนา
มีการตั้งค่าสภาพแวดล้อมการพัฒนา .NET ที่ใช้งานได้บนระบบของคุณ ตรวจสอบว่าคุณได้ติดตั้งเครื่องมือและกรอบงานที่จำเป็นแล้ว
### 4. ความเข้าใจพื้นฐานเกี่ยวกับ C# และ .NET
ทำความคุ้นเคยกับภาษาการเขียนโปรแกรม C# และพื้นฐานของ .NET framework เพื่อปฏิบัติตามบทช่วยสอนได้อย่างมีประสิทธิภาพ

## นำเข้าเนมสเปซ
หากต้องการแสดงเอกสารด้วยแบบอักษรที่กำหนดเองโดยใช้ GroupDocs.Viewer สำหรับ .NET คุณจำเป็นต้องนำเข้าเนมสเปซที่จำเป็นลงในโปรเจ็กต์ของคุณ

```csharp
using System;
using System.IO;
using GroupDocs.Viewer.Fonts;
using GroupDocs.Viewer.Options;
```

## ขั้นตอนที่ 1: ตั้งค่าแหล่งที่มาของแบบอักษร
ขั้นแรก ให้กำหนดแหล่งที่มาของแบบอักษรที่จะใช้ในการแสดงเอกสาร ขั้นตอนนี้จะช่วยให้ GroupDocs.Viewer สามารถเข้าถึงแบบอักษรที่กำหนดเองได้
```csharp
FontSettings.SetFontSources(
    new FolderFontSource(Utils.FontsPath, Fonts.SearchOption.TopFolderOnly));
```
## ขั้นตอนที่ 2: กำหนดไดเรกทอรีผลลัพธ์
ระบุไดเร็กทอรีที่คุณต้องการบันทึกเอกสารที่แสดงผล
```csharp
string outputDirectory = "Your Document Directory";
```
## ขั้นตอนที่ 3: กำหนดรูปแบบเส้นทางไฟล์เพจ
กำหนดรูปแบบการตั้งชื่อไฟล์ HTML เอาท์พุตที่มีหน้าเอกสารที่แสดงผล
```csharp
string pageFilePathFormat = Path.Combine(outputDirectory, "page_{0}.html");
```
## ขั้นตอนที่ 4: เรนเดอร์เอกสารด้วยแบบอักษรที่กำหนดเอง
ใช้ GroupDocs.Viewer API เพื่อแสดงเอกสารด้วยแบบอักษรที่กำหนดเอง แทนที่ `TestFiles.MISSING_FONT_ODG` พร้อมเส้นทางไปยังเอกสารของคุณ
```csharp
using (Viewer viewer = new Viewer(TestFiles.MISSING_FONT_ODG))
{
    HtmlViewOptions options = HtmlViewOptions.ForEmbeddedResources(pageFilePathFormat);
    viewer.View(options);
}
```
## ขั้นตอนที่ 5: แสดงไดเรกทอรีผลลัพธ์
แจ้งให้ผู้ใช้ทราบเกี่ยวกับตำแหน่งที่บันทึกหน้าเอกสารที่แสดงผล
```csharp
Console.WriteLine($"\nSource document rendered successfully.\nCheck output in {outputDirectory}.");
```
