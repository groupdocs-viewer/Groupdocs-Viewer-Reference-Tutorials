## การตั้งค่า GroupDocs.Viewer สำหรับ Java
เริ่มต้นด้วยการตั้งค่า GroupDocs.Viewer ขั้นตอนนี้ง่ายมาก ไม่ว่าคุณจะใช้รุ่นทดลองใช้งานฟรีหรือซื้อไลบรารีแล้วก็ตาม

**การขอใบอนุญาต**
1. **ทดลองใช้งานฟรี:** ดาวน์โหลดใบอนุญาตชั่วคราวเพื่อสำรวจคุณสมบัติเต็มรูปแบบโดยไม่มีข้อจำกัด

3. **ซื้อใบอนุญาต:** สำหรับการใช้งานเชิงพาณิชย์ ให้ซื้อใบอนุญาตผ่าน [ลิงค์นี้](https://purchase-groupdocs.com/buy).

**การเริ่มต้นและการตั้งค่าเบื้องต้น**
เมื่อคุณเพิ่ม GroupDocs.Viewer เป็นส่วนที่ต้องมีในโครงการ Maven แล้ว ให้เริ่มต้นการใช้งานตัวดู:

```java
import com.groupdocs.viewer.Viewer;
import com.groupdocs.viewer.options.HtmlViewOptions;

public class RenderDocumentToHTML {
    public static void main(String[] args) {
        // กำหนดไดเรกทอรีเอาท์พุตสำหรับไฟล์ที่เรนเดอร์
        String outputDirectoryPath = "YOUR_OUTPUT_DIRECTORY/RenderedHTML";
        String pageFilePathFormat = outputDirectoryPath + "/page_{0}.html";

        HtmlViewOptions viewOptions = HtmlViewOptions.forEmbeddedResources(pageFilePathFormat);

        try (Viewer viewer = new Viewer("YOUR_DOCUMENT_DIRECTORY/SAMPLE_DOCX")) {
            viewer.view(viewOptions);
        }
    }
}
```
### คำอธิบาย
- **ตัวเลือก HTMLView:** ระบุว่าควรจัดรูปแบบผลลัพธ์ HTML อย่างไร โดยใช้ `forEmbeddedResources` ทำให้แน่ใจว่าทรัพยากรทั้งหมด เช่น รูปภาพ จะถูกฝังไว้โดยตรงภายในไฟล์ HTML
- **การเริ่มต้นใช้งานตัวแสดง:** เริ่มการทำงานของวัตถุ Viewer ด้วยเส้นทางไปยังไฟล์ DOCX ของคุณ โดยจัดการการล้างทรัพยากรโดยอัตโนมัติ

## คู่มือการใช้งาน
มาแปลงเอกสาร Word เป็นรูปแบบ HTML ที่สามารถเข้าถึงได้ง่าย:

### ขั้นตอนที่ 1: กำหนดไดเรกทอรีผลลัพธ์
```java
String outputDirectoryPath = "YOUR_OUTPUT_DIRECTORY/RenderedHTML";
```
ระบุตำแหน่งที่จะบันทึกไฟล์ที่คุณเรนเดอร์

### ขั้นตอนที่ 2: ตั้งค่ารูปแบบเส้นทางไฟล์เพจ
```java
String pageFilePathFormat = outputDirectoryPath + "/page_{0}.html";
```
การ `{0}` ตัวแทนแสดงหมายเลขลำดับสำหรับการแบ่งหน้า

### ขั้นตอนที่ 3: กำหนดค่า HtmlViewOptions
```java
HtmlViewOptions viewOptions = HtmlViewOptions.forEmbeddedResources(pageFilePathFormat);
```
`forEmbeddedResources` ทำให้แน่ใจว่าทรัพยากรทั้งหมดถูกฝังไว้ใน HTML ทำให้เป็นอิสระและง่ายต่อการแจกจ่ายหรือแสดงออนไลน์

### ขั้นตอนที่ 4: เรนเดอร์เอกสารโดยใช้โปรแกรมดู
การใช้คำสั่ง try-with-resources จะช่วยจัดการทรัพยากรอย่างมีประสิทธิภาพ:
```java
try (Viewer viewer = new Viewer("YOUR_DOCUMENT_DIRECTORY/SAMPLE_DOCX")) {
    viewer.view(viewOptions);
}
```
ตัวอย่างโค้ดนี้จะเปิดไฟล์ DOCX แปลงเป็น HTML โดยใช้ตัวเลือกที่ระบุ และปิดไฟล์โดยอัตโนมัติ `Viewer` ทรัพยากร.

### เคล็ดลับการแก้ไขปัญหา
- **ปัญหาเส้นทางไฟล์:** ตรวจสอบให้แน่ใจว่าเส้นทางทั้งหมดได้รับการตั้งค่าอย่างถูกต้องตามไดเร็กทอรีโครงการของคุณ
- **ความขัดแย้งของเวอร์ชันไลบรารี:** ตรวจสอบให้แน่ใจว่าคุณกำลังใช้ GroupDocs.Viewer เวอร์ชันที่เข้ากันได้กับการตั้งค่า Java ของคุณ
