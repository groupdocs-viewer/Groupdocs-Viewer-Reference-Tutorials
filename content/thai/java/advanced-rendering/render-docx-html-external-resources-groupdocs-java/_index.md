# แปลง DOCX เป็น HTML พร้อมทรัพยากรภายนอกโดยใช้ GroupDocs.Viewer สำหรับ Java

การแปลงเอกสาร DOCX ของคุณเป็น HTML พร้อมคงรักษาทรัพยากรภายนอกเช่นรูปภาพ, Stylesheet, และฟอนต์อาจเป็นเรื่องท้าทาย ด้วย **GroupDocs.Viewer สำหรับ Java** การเรนเดอร์เอกสารเป็นรูปแบบ HTML ที่รวมทรัพยากรที่จำเป็นทั้งหมดจะเป็นเรื่องราบรื่น ฟีเจอร์นี้เป็นประโยชน์อย่างยิ่งเมื่อคุณต้องการให้การนำเสนอคงที่บนหลายแพลตฟอร์ม

![แปลง DOCX เป็น HTML พร้อมทรัพยากรภายนอกด้วย GroupDocs.Viewer สำหรับ Java](/viewer/advanced-rendering/convert-docx-to-html-with-external-resources-java.png)

ในบทแนะนำนี้ คุณจะได้เรียนรู้วิธีใช้ GroupDocs.Viewer สำหรับ Java เพื่อเรนเดอร์ไฟล์ DOCX เป็น HTML พร้อมทรัพยากรภายนอกอย่างมีประสิทธิภาพ เมื่อจบคู่มือแล้ว คุณจะเข้าใจ:
- วิธีตั้งค่าและกำหนดค่า GroupDocs.Viewer สำหรับ Java
- ขั้นตอนที่จำเป็นในการแปลงเอกสาร DOCX เป็นรูปแบบ HTML โดยใช้ทรัพยากรภายนอก
- แนวทางปฏิบัติที่ดีที่สุดสำหรับการเพิ่มประสิทธิภาพการทำงานและการจัดการหน่วยความจำใน Java

## คำตอบสั้น
- **“convert docx to html” หมายถึงอะไร?** มันแปลงไฟล์ Microsoft Word ให้เป็นหน้า HTML ที่เป็นมิตรกับเว็บพร้อมคงรูปภาพ, สไตล์, และฟอนต์ไว้ครบถ้วน  
- **ไลบรารีใดรับหน้าที่แปลง?** GroupDocs.Viewer สำหรับ Java ให้ API ระดับสูงที่ซ่อนการพาร์สระดับล่างไว้  
- **ต้องมีลิขสิทธิ์หรือไม่?** สามารถใช้รุ่นทดลองฟรีเพื่อประเมินได้ แต่ต้องมีลิขสิทธิ์ถาวรสำหรับการใช้งานในผลิตภัณฑ์  
- **สามารถดึงรูปภาพจาก docx ระหว่างการแปลงได้หรือไม่?** ได้ – โหมด external‑resources จะบันทึกรูปภาพแต่ละไฟล์เป็นไฟล์แยก  
- **กระบวนการนี้ประหยัดหน่วยความจำหรือไม่?** การใช้ try‑with‑resources และการสตรีมทำให้การใช้หน่วยความจำต่ำ แม้กับเอกสารขนาดใหญ่

วลีนี้อธิบายกระบวนการนำไฟล์ DOCX (Word) มาสร้างเป็นการแสดงผล HTML ที่เทียบเท่า ซึ่งมีประโยชน์เมื่อคุณต้องการแสดงเนื้อหา Word บนเบราว์เซอร์, ฝังในแอปพลิเคชันเว็บ, หรือเก็บเป็นรูปแบบที่อ่านได้ทั่วโลก

## คู่มือการนำไปใช้

เราจะแบ่งการนำไปใช้เป็นส่วนที่ชัดเจนดังนี้:

### การเรนเดอร์เอกสารพร้อมทรัพยากรภายนอก
ฟีเจอร์นี้ช่วยให้คุณแปลงไฟล์ DOCX เป็น HTML พร้อมคงทรัพยากรภายนอกเช่นรูปภาพแยกไว้แต่เข้าถึงได้

#### ขั้นตอนแบบละเอียด
1. **กำหนดไดเรกทอรีผลลัพธ์และรูปแบบไฟล์**  
   ตั้งค่าพาธสำหรับเก็บไฟล์ผลลัพธ์ รวมถึงรูปแบบการตั้งชื่อสำหรับหน้าและทรัพยากร:

```java
String outputDirectory = "YOUR_OUTPUT_DIRECTORY/RenderToHtmlWithExternalResources";
String pageFilePathFormat = outputDirectory + "/page_{0}.html"; // Naming pattern for HTML pages
String resourceFilePathFormat = outputDirectory + "/page_{0}_{1}"; // Pattern for resources (e.g., images)
String resourceUrlFormat = outputDirectory + "/page_{0}_{1}"; // URL format in generated HTML
```

2. **กำหนด HtmlViewOptions**  
   บอกให้ Viewer สร้างทรัพยากรภายนอกโดยใช้พาธที่คุณกำหนดไว้:

```java
HtmlViewOptions viewOptions = HtmlViewOptions.forExternalResources(pageFilePathFormat, resourceFilePathFormat, resourceUrlFormat);
```

3. **เริ่มต้นและเรนเดอร์เอกสาร**  
   ใช้คลาส `Viewer` เพื่อประมวลผลไฟล์ DOCX ตามตัวเลือกที่ตั้งไว้ข้างต้น:

```java
try (Viewer viewer = new Viewer(TestFiles.SAMPLE_DOCX)) {
    viewer.view(viewOptions); // Renders DOCX as HTML with external resources
}
```

#### ตัวเลือกการกำหนดค่าที่สำคัญ
- **`HtmlViewOptions.forExternalResources()`** – ให้คุณควบคุมตำแหน่งที่หน้า HTML และทรัพยากรจะถูกเขียนและวิธีอ้างอิงใน markup ที่สร้างขึ้น  
- ตัวแปรแทนที่ (`{0}`, `{1}`) จะถูกแทนที่ในเวลารันด้วยหมายเลขหน้าและตัวระบุทรัพยากร เพื่อให้แต่ละไฟล์มีชื่อเฉพาะ

### เคล็ดลับการแก้ปัญหา
- ตรวจสอบว่าไดเรกทอรีผลลัพธ์มีอยู่และแอปมีสิทธิ์เขียน  
- ตรวจสอบรูปแบบ URL อีกครั้ง; URL ที่ไม่ตรงกันจะทำให้ลิงก์รูปภาพเสียใน HTML สุดท้าย  
- ใช้ `try‑catch` เพื่อบันทึกข้อยกเว้นรอบการสร้าง `Viewer` เพื่อวินิจฉัยปัญหาลิขสิทธิ์หรือการเข้าถึงไฟล์

พิจารณากรณีการใช้งานจริงต่อไปนี้:
1. **ระบบจัดการเนื้อหาเว็บ (CMS):** แปลงบทความที่สร้างด้วย Word ให้เป็น HTML พร้อมรูปภาพและสไตล์โดยอัตโนมัติ

การผสานรวมสามารถทำได้กับระบบเช่นแพลตฟอร์ม CMS เพื่ออัปเดตเนื้อหาอย่างต่อเนื่อง

## คำถามที่พบบ่อยเพิ่มเติม

**ถาม:** สามารถดึงรูปภาพจาก docx โดยไม่ต้องแปลงเอกสารทั้งหมดได้หรือไม่?  
**ตอบ:** ได้. โหมด external‑resources จะบันทึกรูปภาพแต่ละไฟล์เป็นไฟล์แยกที่คุณสามารถใช้ได้โดยอิสระ  

**ถาม:** การแปลงจะคงฟอนต์ที่กำหนดเองไว้หรือไม่?  
**ตอบ:** GroupDocs.Viewer จะฝังข้อมูลฟอนต์เมื่อทำได้; หากไม่สามารถทำได้จะใช้ฟอนต์เว็บ‑เซฟเป็นค่าเริ่มต้น  

**ถาม:** HTML ที่สร้างขึ้นรองรับการตอบสนองต่ออุปกรณ์ (responsive) หรือไม่?  
**ตอบ:** HTML จะรักษโครงสร้างต้นฉบับ; คุณสามารถเพิ่ม CSS ของคุณเองเพื่อทำให้ตอบสนองต่ออุปกรณ์ได้  

**ถาม:** ต้องใช้ Java เวอร์ชันใด?  
**ตอบ:** รองรับ Java 8 ขึ้นไป; แนะนำให้ใช้เวอร์ชัน LTS ล่าสุด  

**ถาม:** จะผสานผลลัพธ์กับแอปพลิเคชัน Spring Boot อย่างไร?  
**ตอบ:** ให้บริการไฟล์ HTML และโฟลเดอร์ทรัพยากรที่สร้างขึ้นเป็นคอนเทนต์สแตติกผ่านไดเรกทอรี `resources/static` ของ Spring
