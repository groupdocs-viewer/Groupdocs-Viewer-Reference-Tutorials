# เรนเดอร์เอกสารจาก FTP โดยใช้ GroupDocs.Viewer สำหรับ Java: คู่มือฉบับสมบูรณ์

## การแนะนำ

การเรนเดอร์เอกสารโดยตรงจากเซิร์ฟเวอร์ FTP จะช่วยเพิ่มประสิทธิภาพกระบวนการเวิร์กโฟลว์ได้อย่างมาก โดยเฉพาะในแอปพลิเคชันการเรนเดอร์เอกสารบนคลาวด์และระยะไกล บทช่วยสอนนี้จะแนะนำคุณเกี่ยวกับขั้นตอนในการโหลดและเรนเดอร์เอกสารเป็น HTML โดยใช้ **โปรแกรมดูเอกสารกลุ่ม** ใน Java โดยใช้ประโยชน์จากไลบรารีที่มีประสิทธิภาพนี้สำหรับงานการดูเอกสารที่มีประสิทธิภาพ

### สิ่งที่คุณจะได้เรียนรู้

- เชื่อมต่อกับเซิร์ฟเวอร์ FTP และดึงไฟล์อย่างมีประสิทธิภาพ
- เรนเดอร์เอกสารเป็น HTML โดยใช้ GroupDocs.Viewer สำหรับ Java
- กำหนดค่าตัวเลือกมุมมอง HTML ที่มีทรัพยากรที่ฝังไว้สำหรับผลลัพธ์ที่ได้รับการเพิ่มประสิทธิภาพ
- จัดการข้อยกเว้นอย่างสง่างามและเพิ่มประสิทธิภาพการทำงานอย่างมีประสิทธิผล

มาเริ่มต้นด้วยการกำหนดข้อกำหนดเบื้องต้นที่จำเป็นสำหรับบทช่วยสอนนี้กันก่อน!

## คู่มือการใช้งาน

### คุณสมบัติ 1: การโหลดเอกสารจาก FTP

#### ภาพรวม
ฟีเจอร์นี้สาธิตวิธีการสร้างการเชื่อมต่อกับเซิร์ฟเวอร์ FTP และเรียกค้นเอกสารเป็นสตรีมอินพุต ซึ่งสามารถใช้สำหรับการแสดงผลได้

#### ขั้นตอนการดำเนินการ

##### เชื่อมต่อกับเซิร์ฟเวอร์ FTP

```java
import org.apache.commons.net.ftp.FTPClient;

private static InputStream getFileFromFtp(String server, String filePath) {
    try (FTPClient client = new FTPClient()) { // ปิด FTPClient โดยอัตโนมัติเมื่อเสร็จสิ้น
        client.connect(server);                // เชื่อมต่อกับเซิร์ฟเวอร์ FTP
        return client.retrieveFileStream(filePath); // ดึงข้อมูลไฟล์เป็นสตรีมอินพุต
    } catch (Exception e) {
        throw new RuntimeException(e);       // จัดการข้อยกเว้นโดยการโยนข้อยกเว้นรันไทม์
    }
}
```

- **พารามิเตอร์**- `server` คือที่อยู่เซิร์ฟเวอร์ FTP และ `filePath` ระบุเส้นทางไฟล์บนเซิร์ฟเวอร์
- **ค่าส่งคืน**: วิธีการส่งคืน `InputStream` ของไฟล์ที่ระบุ

### คุณสมบัติ 2: การเรนเดอร์เอกสารจากสตรีม FTP

#### ภาพรวม
คุณลักษณะนี้มุ่งเน้นที่การเรนเดอร์เอกสารที่ได้รับจากสตรีม FTP เป็น HTML โดยใช้ GroupDocs.Viewer สำหรับ Java

#### ขั้นตอนการดำเนินการ

##### กำหนดค่าตัวเลือกเอาท์พุตและมุมมอง

```java
import com.groupdocs.viewer.Viewer;
import com.groupdocs.viewer.options.HtmlViewOptions;

public class RenderDocumentFromFtpStream {
    public static void render() {
        Path outputDirectory = Paths.get("YOUR_OUTPUT_DIRECTORY");
        Path pageFilePathFormat = outputDirectory.resolve("page_{0}.html");

        String server = "localhost";
        String filePath = "sample.doc";

        HtmlViewOptions viewOptions = HtmlViewOptions.forEmbeddedResources(pageFilePathFormat);

        try (InputStream documentStream = getFileFromFtp(server, filePath)) {
            try (Viewer viewer = new Viewer(documentStream)) {
                viewer.view(viewOptions);
            }
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
}
```

- **พารามิเตอร์**- `outputDirectory` ระบุตำแหน่งที่จะบันทึกไฟล์ HTML `pageFilePathFormat` จัดรูปแบบเส้นทางไฟล์ของแต่ละหน้า
- **ตัวเลือกการกำหนดค่าคีย์**:การใช้ทรัพยากรที่ฝังไว้ทำให้แน่ใจว่าทรัพยากรที่เกี่ยวข้องทั้งหมดจะรวมอยู่ในเอาต์พุต HTML

#### เคล็ดลับการแก้ไขปัญหา

- ตรวจสอบให้แน่ใจว่าสามารถเข้าถึงเซิร์ฟเวอร์ FTP ได้ และข้อมูลประจำตัวหากจำเป็น ได้รับการกำหนดค่าอย่างถูกต้อง
- ตรวจสอบว่าเส้นทางไฟล์ที่ระบุบนเซิร์ฟเวอร์ FTP ตรงกับเส้นทางที่ใช้ในโค้ด
- ตรวจสอบข้อยกเว้นระหว่างการดำเนินการสตรีมเพื่อแก้ไขปัญหาการเชื่อมต่ออย่างมีประสิทธิภาพ

3. **เครื่องมือการทำงานร่วมกัน**:อำนวยความสะดวกในการดูเอกสารในรูปแบบที่สอดคล้องกันระหว่างสมาชิกในทีมไม่ว่าจะอยู่ที่ใดก็ตาม
