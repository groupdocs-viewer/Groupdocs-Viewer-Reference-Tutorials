# วิธีการแสดงผล HTML และยกเว้นฟอนต์ Arial ด้วย GroupDocs.Viewer Java

การแปลงเอกสารเป็น HTML เป็นความต้องการทั่วไปสำหรับแอปพลิเคชันบนเว็บ, แต่ฟอนต์ที่ไม่ต้องการอาจทำให้ความสอดคล้องของภาพเสียหาย ในบทแนะนำนี้ คุณจะได้เรียนรู้ **วิธีการแสดงผล html** พร้อมยกเว้นฟอนต์ Arial เพื่อให้ผลลัพธ์ตรงกับแนวทางการออกแบบของคุณ เราจะครอบคลุมวิธี **แปลง docx เป็น html**, ฝังทรัพยากรในหน้าที่สร้างขึ้น, และ **เพิ่มประสิทธิภาพการแสดงผล html** เพื่อประสิทธิภาพที่ดียิ่งขึ้น.

![ยกเว้นฟอนต์ Arial ในการแสดงผล HTML ด้วย GroupDocs.Viewer สำหรับ Java](/viewer/custom-rendering/exclude-arial-font-in-html-rendering-java.png)

**สิ่งที่คุณจะได้เรียนรู้:**
- วิธีการกำหนดค่า GroupDocs.Viewer สำหรับ Java เพื่อแสดงผลเอกสารในรูปแบบ HTML.
- กระบวนการยกเว้นฟอนต์เฉพาะเช่น Arial ระหว่างการแปลงเอกสาร.
- แนวปฏิบัติที่ดีที่สุดและการพิจารณาด้านประสิทธิภาพเมื่อใช้ GroupDocs.Viewer Java.

## คำตอบอย่างรวดเร็ว
- **วิธีการแสดงผล html?** ใช้ `HtmlViewOptions` กับ GroupDocs.Viewer Java เพื่อสร้างหน้า HTML ที่เป็นอิสระ.  
- **ฉันสามารถยกเว้นฟอนต์ Arial ได้หรือไม่?** ได้—เรียก `viewOptions.getFontsToExclude().add("Arial")`.  
- **เวอร์ชันของไลบรารีคืออะไร?** คู่มือนี้ใช้ GroupDocs.Viewer for Java 25.2 (หรือรุ่นเสถียรล่าสุด).  
- **รูปแบบไฟล์อินพุตที่รองรับคืออะไร?** DOCX, PDF, PPTX, XLSX, และอื่น ๆ อีกมาก.  
- **จำเป็นต้องมีลิขสิทธิ์หรือไม่?** การทดลองใช้ฟรีเพียงพอสำหรับการทดสอบ; จำเป็นต้องมีลิขสิทธิ์เต็มสำหรับการใช้งานจริง.

## วิธีการยกเว้นฟอนต์ Arial เมื่อแสดงผล HTML

### ภาพรวม
การยกเว้นฟอนต์เฉพาะช่วยให้คุณควบคุมผลลัพธ์ภาพของการแปลง HTML ได้ดีขึ้น ช่วยให้คุณ **เพิ่มประสิทธิภาพการแสดงผล html** เพื่อความเร็วและความสอดคล้องของแบรนด์.

### การดำเนินการแบบขั้นตอน

#### 1. กำหนดไดเรกทอรีเอาต์พุต
เริ่มต้นโดยระบุที่เก็บไฟล์ HTML:

```java
Path outputDirectory = Path.of("YOUR_OUTPUT_DIRECTORY");
```

เส้นทางนี้กำหนดตำแหน่งที่เอกสาร HTML ที่แปลงแล้วจะถูกเก็บไว้.

#### 2. ตั้งค่าเส้นทางไฟล์หน้า HTML
กำหนดรูปแบบชื่อไฟล์ของแต่ละหน้าดังนี้:

```java
Path pageFilePathFormat = outputDirectory.resolve("page_{0}.html");
```

ตัวแปร `{0}` ช่วยให้ตั้งชื่อไฟล์แบบไดนามิกตามหมายเลขหน้า เพื่อให้การจัดเก็บเป็นระเบียบ.

#### 3. กำหนดค่า View Options พร้อมฝังทรัพยากร
สร้างอ็อบเจกต์ `HtmlViewOptions` ที่ระบุวิธีการจัดการการแสดงผล HTML:

```java
HtmlViewOptions viewOptions = HtmlViewOptions.forEmbeddedResources(pageFilePathFormat);
```

การตั้งค่านี้ทำให้ทรัพยากรทั้งหมดฝังอยู่ในไฟล์ HTML ทำให้ไฟล์เป็น **self‑contained** และเหมาะสำหรับสถานการณ์ **embed resources html**.

#### 4. ยกเว้นฟอนต์เฉพาะ
เพิ่มฟอนต์ที่ต้องการยกเว้น (ในกรณีนี้คือ Arial) จากการแสดงผลในเอาต์พุต:

```java
viewOptions.getFontsToExclude().add("Arial");
```

การยกเว้นฟอนต์อาจสำคัญต่อการรักษาความสอดคล้องของการออกแบบและลดขนาดไฟล์.

#### 5. แสดงผลเอกสารโดยใช้ Viewer
สุดท้าย ใช้คลาส `Viewer` เพื่อแปลงเอกสารของคุณเป็นรูปแบบ HTML:

```java
try (Viewer viewer = new Viewer("YOUR_DOCUMENT_DIRECTORY/SAMPLE_DOCX")) {
    viewer.view(viewOptions);
}
```

คำสั่ง try‑with‑resources ทำให้แน่ใจว่า `viewer` ถูกปิดอย่างถูกต้องหลังการแปลง.

### เคล็ดลับการแก้ไขปัญหา
- **ปัญหาทั่วไป**: ตรวจสอบให้แน่ใจว่าเส้นทางถูกต้องและเข้าถึงได้; เส้นทางที่ไม่ถูกต้องอาจทำให้เกิดข้อผิดพลาดไฟล์ไม่พบ.
- **เคล็ดลับประสิทธิภาพ**: เมื่อแปลงเอกสารขนาดใหญ่ ให้ตรวจสอบการใช้หน่วยความจำ เนื่องจากการฝังทรัพยากรอาจทำให้เวลาโหลดเพิ่มขึ้น.

## วิธีการแปลง DOCX เป็น HTML ด้วย GroupDocs.Viewer
การกำหนดค่า `HtmlViewOptions` เดียวกันทำงานกับรูปแบบที่รองรับทั้งหมด รวมถึง DOCX เพียงชี้ตัวสร้าง `Viewer` ไปที่ไฟล์ `.docx` แล้วไลบรารีจะจัดการการแปลงพร้อมคำนึงถึงการตั้งค่าการยกเว้นฟอนต์.

## ทำไมเรื่องนี้สำคัญ: กรณีการใช้งานจริง

1. **รายงานองค์กร** – ยกเว้นฟอนต์เริ่มต้นเพื่อให้รายงานสอดคล้องกับแนวทางแบรนด์.  
2. **สื่อการศึกษา** – ปรับแต่งการแสดงผลฟอนต์เพื่อความอ่านง่ายบนอุปกรณ์ต่าง ๆ.  
3. **เอกสารทางกฎหมาย** – รักษาลักษณะที่สอดคล้องสำหรับการนำเสนอ HTML ที่พร้อมใช้ในศาล.  
4. **รายการสินค้าอีคอมเมิร์ซ** – ทำให้คำอธิบายสินค้าตรงตามมาตรฐานแบรนด์.  
5. **การผสานรวม CMS** – ให้การแสดงตัวอย่างที่สะอาดและควบคุมฟอนต์ภายในระบบจัดการเนื้อหา.

## ปรับประสิทธิภาพการแสดงผล HTML

### เคล็ดลับเพื่อการแปลงที่เร็วขึ้น
- **ใช้เส้นทางไฟล์ที่มีประสิทธิภาพ**: ทำให้โครงสร้างไดเรกทอรีตื้นเพื่อ ลดภาระ I/O.  
- **จำกัดทรัพยากรที่ฝัง**: ฝังเฉพาะ CSS/JS ที่จำเป็นเพื่อให้ HTML มีน้ำหนักเบา.

### แนวปฏิบัติที่ดีที่สุดสำหรับการจัดการหน่วยความจำใน Java
- **ปิด Viewer อย่างรวดเร็ว**: รูปแบบ `try‑with‑resources` จะปล่อยหน่วยความจำทันทีที่การแปลงเสร็จ.  
- **ตรวจสอบโหลดของแอปพลิเคชัน**: ทำการ profiling JVM เมื่อจัดการหลายเอกสารหรือเอกสารขนาดใหญ่เพื่อหลีกเลี่ยงข้อผิดพลาด out‑of‑memory.
