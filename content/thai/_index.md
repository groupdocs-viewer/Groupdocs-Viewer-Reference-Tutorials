# GroupDocs.Viewer บทเรียน

ยินดีต้อนรับสู่ศูนย์บทเรียนของ GroupDocs.Viewer ที่นี่คุณจะพบกับคอลเลกชันบทเรียนและคู่มือที่ครอบคลุมเพื่อช่วยคุณ **add watermark document** และเชี่ยวชาญ API ตัวดูเอกสารที่ทรงพลังของเราสำหรับ .NET และ Java ไม่ว่าคุณจะสร้างแอปพลิเคชันเว็บ แอปเดสก์ท็อป หรือโซลูชันมือถือ GroupDocs.Viewer จะทำให้คุณเรนเดอร์และแสดงเอกสารหลากหลายรูปแบบได้อย่างไร้รอยต่อ รวมถึง PDF, Microsoft Office (Word, Excel, PowerPoint), CAD, รูปภาพ และอื่น ๆ อีกมากมาย

## คำตอบด่วน
- **“add watermark document” หมายถึงอะไร?** หมายถึงการฝังข้อความหรือรูปภาพกึ่งโปร่งใสเป็นชั้นทับบนแต่ละหน้าของเอกสารที่เรนเดอร์  
- **ฟอร์แมตใดบ้างที่รองรับวอเตอร์มาร์ก?** ทุกฟอร์แมตที่ GroupDocs.Viewer เรนเดอร์ได้ รวมถึง PDF, DOCX, XLSX, CAD, และข้อความอีเมล  
- **ต้องมีลิขสิทธิ์หรือไม่?** จำเป็นต้องมีลิขสิทธิ์ GroupDocs.Viewer ที่ถูกต้องสำหรับการใช้งานในผลิตภัณฑ์; มีรุ่นทดลองฟรีให้ใช้  
- **สามารถรวมวอเตอร์มาร์กกับฟีเจอร์อื่นได้หรือไม่?** ได้—วอเตอร์มาร์กสามารถใช้ร่วมกับการแคช, การเรนเดอร์แบบกำหนดเอง, และการตั้งค่าความปลอดภัยได้  
- **รองรับ .NET Core หรือไม่?** แน่นอน—GroupDocs.Viewer ทำงานกับ .NET Framework, .NET Core, และ .NET 5/6+

## วิธีเพิ่ม Watermark Document ด้วย GroupDocs.Viewer

การเพิ่มวอเตอร์มาร์กมักจำเป็นสำหรับการสร้างแบรนด์, ความลับ, หรือการปฏิบัติตามกฎระเบียบ ด้วย GroupDocs.Viewer คุณสามารถใส่วอเตอร์มาร์กขณะเรนเดอร์เอกสารเป็น HTML, PDF, หรือรูปภาพ ขั้นตอนง่าย ๆ มีดังนี้:

1. **สร้างอ็อบเจ็กต์ `WatermarkOptions`** แล้วตั้งค่าข้อความ, สี, ความโปร่งใส, และการหมุน  
2. **ส่งอ็อบเจ็กต์นี้** ไปยังเมธอดการเรนเดอร์ที่เหมาะสม (เช่น `RenderToPdf`, `RenderToHtml`, หรือ `RenderToImage`)  
3. **เรนเดอร์เอกสาร** — ผลลัพธ์จะมีวอเตอร์มาร์กบนทุกหน้า

วิธีนี้ทำงานได้กับไฟล์ประเภทที่รองรับทั้งหมด รวมถึงสถานการณ์ **render pdf .net**, ภาพวาด CAD, และข้อความอีเมล

## คำถามที่พบบ่อย

**Q: สามารถเพิ่มวอเตอร์มาร์กให้เอกสารที่เรนเดอร์เป็นรูปภาพได้หรือไม่?**  
A: ได้—ใช้ `WatermarkOptions` ร่วมกับ `RenderToImage` เพื่อฝังวอเตอร์มาร์กบนแต่ละหน้าภาพที่สร้างขึ้น

**Q: การเพิ่มวอเตอร์มาร์กส่งผลต่อประสิทธิภาพการเรนเดอร์หรือไม่?**  
A: ผลกระทบค่อนข้างน้อย; GroupDocs.Viewer ปรับกระบวนการวางชั้นทับให้เหมาะสม โดยเฉพาะเมื่อใช้ร่วมกับการแคช

**Q: วอเตอร์มาร์กรองรับการเรนเดอร์ข้อความอีเมลหรือไม่?**  
A: แน่นอน—วอเตอร์มาร์กสามารถใส่ลงในผลลัพธ์ HTML หรือ PDF ของข้อความอีเมลที่เรนเดอร์ได้

**Q: จะปรับแต่งลักษณะของวอเตอร์มาร์กอย่างไร?**  
A: คุณสามารถตั้งค่าครอบครัวฟอนต์, ขนาด, สี, ความโปร่งใส, มุมการหมุน, และแม้แต่ใช้รูปภาพเป็นวอเตอร์มาร์กได้

**Q: สามารถใช้วอเตอร์มาร์กที่แตกต่างกันในแต่ละหน้าได้หรือไม่?**  
A: API มาตรฐานจะใส่วอเตอร์มาร์กแบบเดียวกันบนทุกหน้า แต่คุณสามารถเขียนโลจิกการเรนเดอร์แบบกำหนดเองเพื่อเปลี่ยนวอเตอร์มาร์กตามหน้าได้
