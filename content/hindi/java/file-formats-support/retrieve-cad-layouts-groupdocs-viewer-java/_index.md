- **क्या लाइसेंस की आवश्यकता है?** एक फ्री ट्रायल उपलब्ध है; उत्पादन उपयोग के लिए व्यावसायिक लाइसेंस आवश्यक है।

- **क्या Maven आवश्यक है?** Maven GroupDocs.Viewer को आपके प्रोजेक्ट में जोड़ने का अनुशंसित तरीका है, लेकिन आप Gradle या मैन्युअल JARs भी उपयोग कर सकते हैं।

## कार्यान्वयन गाइड

नीचे एक चरण‑दर‑चरण walkthrough दिया गया है जो दिखाता है कि GroupDocs.Viewer का उपयोग करके **retrieve CAD layouts Java** कैसे किया जाए।

### चरण 1: Viewer को इनिशियलाइज़ करें
अपने CAD फ़ाइल की ओर इशारा करके एक `Viewer` इंस्टेंस बनाएं। `try‑with‑resources` ब्लॉक सुनिश्चित करता है कि Viewer सही ढंग से बंद हो, जिससे मेमोरी मुक्त हो जाती है।

```java
import com.groupdocs.viewer.Viewer;
import java.io.File;

String documentPath = new File("YOUR_DOCUMENT_DIRECTORY", "SAMPLE_DWG_WITH_LAYOUTS_AND_LAYERS").getAbsolutePath();

try (Viewer viewer = new Viewer(documentPath)) {
    // Further operations will be performed here.
}
```

### चरण 2: व्यू जानकारी प्राप्त करें
`ViewInfoOptions.forHtmlView()` के साथ `getViewInfo` का उपयोग करके एक `CadViewInfo` ऑब्जेक्ट प्राप्त करें जिसमें लेआउट और लेयर कलेक्शन होते हैं।

```java
import com.groupdocs.viewer.options.ViewInfoOptions;
import com.groupdocs.viewer.results.CadViewInfo;

CadViewInfo info = (CadViewInfo) viewer.getViewInfo(ViewInfoOptions.forHtmlView());
```

### चरण 3: लेआउट और लेयर निकालें
`layouts` और `layers` कलेक्शन पर इटरेट करें। आप उन्हें लॉग कर सकते हैं, डेटाबेस में स्टोर कर सकते हैं, या डाउनस्ट्रीम प्रोसेस में फीड कर सकते हैं।

```java
// Iterate over each layout in the CAD file
for (Layout layout : info.getLayouts()) {
    // Process each layout as needed
}

// Iterate over each layer in the CAD file
for (Layer layer : info.getLayers()) {
    // Process each layer as needed
}
```

### सामान्य समस्याएँ और उनके समाधान
- **फ़ाइल नहीं मिली:** `Viewer` को पास किए गए पाथ को दोबारा जांचें। पूर्ण पाथ का उपयोग करें या कार्य निर्देशिका सत्यापित करें।  
- **वर्ज़न मिसमैच:** सुनिश्चित करें कि GroupDocs.Viewer का वर्ज़न आपके JDK से मेल खाता है (25.x सीरीज़ JDK 8‑17 के साथ काम करती है)।  
- **मेमोरी लीक:** हमेशा ऊपर दिखाए गए `try‑with‑resources` पैटर्न का उपयोग करें; यह स्वचालित रूप से नेटिव रिसोर्सेज़ को रिलीज़ करता है।
