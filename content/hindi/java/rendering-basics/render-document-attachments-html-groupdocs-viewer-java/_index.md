# GroupDocs.Viewer Java के साथ HTML में दस्तावेज़ अनुलग्नक प्रस्तुत करें

## परिचय

वेब अनुप्रयोगों में दस्तावेज़ अनुलग्नकों को प्रभावी ढंग से प्रदर्शित करना चुनौतीपूर्ण हो सकता है। **जावा के लिए GroupDocs.Viewer** इन अनुलग्नकों को HTML प्रारूप में सहजता से प्रस्तुत करने के लिए एक मजबूत समाधान प्रदान करता है, जो एम्बेडेड फाइलों के साथ ईमेल जैसे दस्तावेजों को इंटरैक्टिव और दृश्यमान रूप से आकर्षक HTML पृष्ठों में परिवर्तित करता है।

इस ट्यूटोरियल में, आप सीखेंगे कि दस्तावेज़ अनुलग्नकों को रेंडर करके अपने एप्लिकेशन की कार्यक्षमता को बढ़ाने के लिए GroupDocs.Viewer जावा लाइब्रेरी का उपयोग कैसे करें। 

**मुख्य सीखें:**
- Java के लिए GroupDocs.Viewer को सेट अप और आरंभ करें
- दस्तावेज़ों से अनुलग्नकों को HTML में प्रस्तुत करें
- CacheableFactory का उपयोग करके कुशल अनुलग्नक प्रबंधन
- दस्तावेज़ रूपांतरणों को संभालते समय प्रदर्शन को अनुकूलित करें

## कार्यान्वयन मार्गदर्शिका

यह मार्गदर्शिका दस्तावेज़ अनुलग्नकों को HTML में प्रस्तुत करने और CacheableFactory का उपयोग करके इन्हें प्रबंधित करने के बारे में बताती है।

### दस्तावेज़ अनुलग्नकों को HTML में प्रस्तुत करें

किसी दस्तावेज़ से अनुलग्नक को, जैसे ईमेल में एम्बेडेड फ़ाइलें, वेब अनुप्रयोगों में निर्बाध प्रदर्शन के लिए HTML प्रारूप में परिवर्तित करें।

#### अवलोकन
Word दस्तावेज़ों वाले ईमेल जैसे दस्तावेज़ों से अनुलग्नक निकालना सीखें और उन्हें GroupDocs.Viewer के साथ इंटरैक्टिव HTML पृष्ठों के रूप में प्रस्तुत करें।

##### चरण 1: आउटपुट निर्देशिका सेट करें
आउटपुट निर्देशिका को परिभाषित करें जहां रेंडर की गई HTML फ़ाइलें सहेजी जाएंगी:

```java
Path YOUR_OUTPUT_DIRECTORY = Utils.getOutputDirectoryPath("RenderDocumentAttachments");
Path pageFilePathFormat = YOUR_OUTPUT_DIRECTORY.resolve("page_{0}.html");
```

##### चरण 2: एक अनुलग्नक ऑब्जेक्ट बनाएँ
उपयोग `CacheableFactory` बनाने के लिए `Attachment` ऑब्जेक्ट, कुशल कैशिंग में सहायता:

```java
Attachment attachment = CacheableFactory.getInstance().newAttachment("attachment-word.doc", pageFilePathFormat.toString());
```

##### चरण 3: अनुलग्नक को HTML में निकालें और प्रस्तुत करें
उपयोग करें `Viewer` निर्दिष्ट दस्तावेज़ के अनुलग्नक को एम्बेडेड संसाधनों के साथ HTML प्रारूप में प्रस्तुत करने के लिए क्लास:

```java
try (ByteArrayOutputStream attachmentStream = new ByteArrayOutputStream();
     Viewer viewer = new Viewer("YOUR_DOCUMENT_DIRECTORY/SAMPLE_MSG_WITH_ATTACHMENTS")) {
    
    // निर्दिष्ट अनुलग्नक को बाइट स्ट्रीम में सहेजें.
    viewer.saveAttachment(attachment, attachmentStream);

    try (InputStream inputStream = new ByteArrayInputStream(attachmentStream.toByteArray());
         Viewer attachmentViewer = new Viewer(inputStream)) {
        
        HtmlViewOptions options = HtmlViewOptions.forEmbeddedResources(pageFilePathFormat);
        attachmentViewer.view(options); // अनुलग्नक को HTML में प्रस्तुत करें.
    }
} catch (IOException e) {
    throw new RuntimeException(e);
}
```

##### प्रमुख चरणों का स्पष्टीकरण:
- **कैशएबलफैक्ट्री**: अनावश्यक प्रसंस्करण को रोकने के लिए कैशिंग का प्रबंधन करता है, तथा प्रदर्शन को बढ़ाता है।
- **HTMLदृश्यविकल्प**संपूर्ण दृश्य अनुभव के लिए एम्बेडेड संसाधनों के साथ रेंडरिंग को कॉन्फ़िगर करता है।

### अनुलग्नक प्रबंधन के लिए CacheableFactory को आरंभीकृत करें और उसका उपयोग करें

बड़े दस्तावेज़ों या एकाधिक फ़ाइलों से निपटने के दौरान कुशल अनुलग्नक प्रबंधन महत्वपूर्ण है। यह सुविधा उपयोग को प्रदर्शित करती है `CacheableFactory` दस्तावेज़ अनुलग्नकों को संभालने का अनुकूलन करने के लिए।

#### अवलोकन
अपने GroupDocs.Viewer अनुप्रयोगों में बेहतर प्रदर्शन के लिए कैश प्रबंधक को आरंभ करने के लाभों के बारे में जानें।

##### चरण 1: CacheableFactory का उपयोग करके एक अटैचमेंट ऑब्जेक्ट बनाएं

```java
Attachment attachment = CacheableFactory.getInstance().newAttachment("attachment-word.doc", "YOUR_OUTPUT_DIRECTORY/page_{0}.html");
```

#### स्पष्टीकरण:
- **कैशएबलफैक्ट्री**: कुशल कैश प्रबंधन प्रदान करता है, संसाधन उपयोग को कम करता है और गति में सुधार करता है।

## व्यावहारिक अनुप्रयोगों

दस्तावेज़ अनुलग्नकों को HTML में प्रस्तुत करना विभिन्न परिदृश्यों में लाभदायक हो सकता है:

1. **ईमेल क्लाइंट:** अलग से फ़ाइल डाउनलोड की आवश्यकता के बिना ईमेल अनुलग्नकों को सीधे क्लाइंट के भीतर प्रदर्शित करें।
2. **दस्तावेज़ प्रबंधन प्रणालियाँ:** उपयोगकर्ताओं को एकल दस्तावेज़ इंटरफ़ेस से सभी एम्बेडेड फ़ाइलों को सहजता से देखने की अनुमति दें।
3. **वेब पोर्टल:** इंटरैक्टिव तत्वों के साथ व्यापक दस्तावेज़ प्रदर्शित करके उपयोगकर्ता अनुभव को बढ़ाएं।

## प्रदर्शन संबंधी विचार

GroupDocs.Viewer का उपयोग करते समय, इन प्रदर्शन अनुकूलन युक्तियों पर विचार करें:
- कैशिंग तंत्र का उपयोग करें `CacheableFactory` अनावश्यक प्रसंस्करण को न्यूनतम करने के लिए।
- मेमोरी उपयोग पर नज़र रखें और बड़े दस्तावेज़ों को संभालने के लिए अपने एप्लिकेशन को अनुकूलित करें।
- मेमोरी प्रबंधन के लिए जावा की सर्वोत्तम प्रथाओं का पालन करें, जैसे स्ट्रीम का कुशलतापूर्वक उपयोग करना और संसाधनों को तुरंत बंद करना।
