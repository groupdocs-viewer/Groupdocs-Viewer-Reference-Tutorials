## Perguntas Frequentes

**Q: Posso converter DOCX para HTML sem gravar arquivos temporários?**  
A: Sim, a API suporta conversão em memória, permitindo que você transmita o HTML diretamente ao cliente.

**Q: Como a conversão em lote java afeta o desempenho?**  
A: Ao converter muitos documentos em um loop, reutilize uma única instância de `Viewer` e habilite streaming para manter o uso de memória baixo.

**Q: É possível converter documentos para PDF enquanto transmite a saída?**  
A: Absolutamente – a biblioteca fornece conversão de PDF em streaming, que grava o PDF diretamente em um fluxo de saída sem carregar todo o arquivo na memória.

**Q: E se eu precisar converter um PDF grande em imagens?**  
A: Use as opções de paginação para processar uma página por vez e combine streaming com um buffer fixo para evitar erros de falta de memória.

**Q: O GroupDocs.Viewer suporta arquivos protegidos por senha?**  
A: Sim, você pode passar a senha ao carregar o documento, e a conversão prosseguirá de forma segura.
