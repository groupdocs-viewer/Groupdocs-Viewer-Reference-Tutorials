**Руководство по внедрению:**
### Функция 1: Получение базовой информации о документе
#### Обзор:
Получение необходимой информации о документе имеет решающее значение для понимания его структуры перед выполнением операций. GroupDocs.Viewer позволяет извлекать метаданные, такие как количество страниц и тип файла.

**Пошаговая реализация:**
##### Шаг 1: Инициализируйте средство просмотра с вашим документом
Укажите путь к вашему документу, заменив `'YOUR_DOCUMENT_DIRECTORY'` на фактический каталог, где хранятся ваши файлы:
```csharp
string filePath = @"YOUR_DOCUMENT_DIRECTORY\SamplePDF.pdf";
```
##### Шаг 2: Извлечение информации о представлении для HTML-рендеринга
Создать экземпляр `ViewInfoOptions` разработан специально для отображения в формате HTML для эффективного доступа к информации о виде документа:
```csharp
using (Viewer viewer = new Viewer(filePath))
{
    ViewInfoOptions options = ViewInfoOptions.ForHtmlView();
    ViewInfo info = viewer.GetViewInfo(options);
    
    // Вывод базовой информации о документе, например количества страниц.
    Console.WriteLine($"Document type: {info.FileType}");
    Console.WriteLine($"Page count: {info.Pages.Count}");
}
```
**Объяснение:** 
- `ForHtmlView()` настраивает параметры для получения сведений о представлении, подходящих для HTML-рендеринга.
- `GetViewInfo(options)` извлекает метаданные о документе.

##### Советы по устранению неполадок:
- Убедитесь, что путь к файлу указан правильно и доступен приложению.
- Убедитесь, что формат документа поддерживается GroupDocs.Viewer, если возникают ошибки `GetViewInfo`.

### Функция 2: Настройка выходного каталога для представлений документов
#### Обзор:
Пользовательские выходные каталоги необходимы при рендеринге документов в различные форматы. Эта функция позволяет указать, где должны храниться рендеринговые файлы, обеспечивая лучший контроль над организацией файловой системы.

**Пошаговая реализация:**
##### Шаг 1: Определите входные и выходные пути
Настройте переменные для входных (исходный документ) и выходных путей:
```csharp
string filePath = @"YOUR_DOCUMENT_DIRECTORY\SamplePDF.pdf";
string outputPath = @"@YOUR_OUTPUT_DIRECTORY";
```
##### Шаг 2: Инициализация средства просмотра и настройка параметров просмотра HTML
Настроить `HtmlViewOptions` чтобы указать, где следует сохранять обработанные HTML-файлы, используя `{page}.html` в качестве заполнителя для динамического именования:
```csharp
using (Viewer viewer = new Viewer(filePath))
{
    HtmlViewOptions options = HtmlViewOptions.ForEmbeddedResources(outputPath + "\{page}.html");
    viewer.View(options);
}
```
**Объяснение:** 
- `ForEmbeddedResources()` обеспечивает встраивание таких ресурсов, как изображения, в HTML, что упрощает развертывание.
- Указанный путь в `outputPath` имеет решающее значение для эффективной организации выходных файлов.

##### Советы по устранению неполадок:
- Проверьте права доступа к выходному каталогу, чтобы убедиться, что в него можно записывать файлы.
- Проверьте строку формата, используемую для наименования страниц (например, `{page}.html`) для предотвращения ошибок во время выполнения.
