## Руководство по внедрению
### Функция 1: Проверка шифрования файлов
#### Обзор
Эта функция позволяет определить, зашифрован ли документ, гарантируя безопасность конфиденциальных данных.

#### Пошаговая реализация
##### Импорт требуемых классов
Начнем с импорта необходимых классов:
```java
import com.groupdocs.viewer.Viewer;
import com.groupdocs.viewer.results.FileInfo;
```

##### Инициализация средства просмотра и проверка шифрования
Заменять `'YOUR_DOCUMENT_DIRECTORY'` с путем к вашему документу:
```java
String filePath = "YOUR_DOCUMENT_DIRECTORY/ENCRYPTED";
try (Viewer viewer = new Viewer(filePath)) {
    FileInfo fileInfo = viewer.getFileInfo();
    boolean isEncrypted = fileInfo.isEncrypted();

    if(isEncrypted) {
        System.out.println("The file is encrypted.");
    } else {
        System.out.println("The file is not encrypted.");
    }
}
```
- **Параметры и назначение метода:** `viewer.getFileInfo()` извлекает метаданные, включая статус шифрования.
- **Совет по устранению неполадок:** Убедитесь, что путь к документу указан правильно, чтобы избежать `FileNotFoundException`.

### Функция 2: Определение типа файла
#### Обзор
Быстро определяйте тип файла, чтобы соответствующим образом адаптировать этапы обработки.

#### Пошаговая реализация
##### Получить и вывести тип файла
Используйте ту же начальную настройку, что и выше, для импорта классов:
```java
String filePath = "YOUR_DOCUMENT_DIRECTORY/ANY_FILE";
try (Viewer viewer = new Viewer(filePath)) {
    FileInfo fileInfo = viewer.getFileInfo();
    String fileType = fileInfo.getFileType();

    System.out.println("The file type is: " + fileType);
}
```
- **Параметры и назначение метода:** `fileInfo.getFileType()` возвращает формат документа.
- **Совет по устранению неполадок:** Неподдерживаемые файлы могут возвращать нулевой или неожиданный результат.
