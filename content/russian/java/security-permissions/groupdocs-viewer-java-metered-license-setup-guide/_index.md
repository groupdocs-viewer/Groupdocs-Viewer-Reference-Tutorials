## Настройка GroupDocs.Viewer для Java

Для начала включите GroupDocs.Viewer в качестве зависимости в свой проект с помощью Maven:

**Настройка Maven**

```xml
<repositories>
   <repository>
      <id>repository.groupdocs.com</id>
      <name>GroupDocs Repository</name>
      <url>https://releases.groupdocs.com/viewer/java/</url>
   </repository>
</repositories>
<dependencies>
   <dependency>
      <groupId>com.groupdocs</groupId>
      <artifactId>groupdocs-viewer</artifactId>
      <version>25.2</version>
   </dependency>
</dependencies>
```

## Руководство по внедрению

### Установить функцию измеренной лицензии

Эта функция позволяет вам установить лимитированную лицензию с помощью API GroupDocs.Viewer, обеспечивая контролируемое использование и управление расходами.

#### Обзор

The `Metered` класс является частью библиотеки GroupDocs.Viewer. Он позволяет разработчикам настраивать лицензирование на основе показателей использования, таких как количество документов или сеансов пользователей.

#### Этапы внедрения

##### Шаг 1: Определите лицензионные ключи

Начните с определения открытого и закрытого ключей для лимитной лицензии:

```java
String publicKey = "your-public-key";
String privateKey = "your-private-key";
```

Заменять `"your-public-key"` и `"your-private-key"` с фактическими значениями, предоставленными GroupDocs при приобретении вами лимитированной лицензии.

##### Шаг 2: Инициализация измеряемого экземпляра

Создайте экземпляр `Metered` сорт:

```java
Metered metered = new Metered();
```

##### Шаг 3: Установите лицензионные ключи

Используйте `setMeteredKey` Метод применения ваших открытого и закрытого ключей:

```java
metered.setMeteredKey(publicKey, privateKey);
```

Этот шаг имеет решающее значение, поскольку он связывает ваше приложение с сервером лицензирования GroupDocs для отслеживания использования.

#### Советы по устранению неполадок

- Убедитесь, что ваши ключи правильно отформатированы и действительны.
- Проверьте сетевое подключение, если у вас возникли проблемы при удаленной настройке лицензии.
- Ознакомьтесь с документацией GroupDocs на предмет изменений или обновлений, связанных с конкретной версией.
