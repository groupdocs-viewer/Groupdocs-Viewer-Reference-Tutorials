### Руководство по внедрению

Теперь давайте сосредоточимся на реализации рендеринга PDF с сохранением исходного размера страницы. Мы подробно проведем вас через каждый шаг.

#### Инициализировать GroupDocs.Viewer
**Обзор:**
Начните с настройки `Viewer` экземпляр для вашего исходного документа.

```java
import com.groupdocs.viewer.Viewer;
import com.groupdocs.viewer.options.PngViewOptions;

import java.nio.file.Path;

public class RenderOriginalPageSize {
    public static void main(String[] args) {
        // Определить путь к выходному каталогу для визуализированных страниц
        Path outputDirectory = Path.of("YOUR_OUTPUT_DIRECTORY");
        
        // Формат путей к выходным файлам страниц
        String pageFilePathFormat = "page_{0}.png";
        Path pageFilePath = outputDirectory.resolve(pageFilePathFormat);
        
        // Инициализируйте PngViewOptions с форматом пути
        PngViewOptions viewOptions = new PngViewOptions(pageFilePath.toString());
        
        // Установить параметр для отображения исходного размера страницы для документов PDF
        viewOptions.getPdfOptions().setRenderOriginalPageSize(true);
        
        // Создать экземпляр Viewer для исходного PDF-документа
        try (Viewer viewer = new Viewer("YOUR_DOCUMENT_DIRECTORY/SAMPLE_PDF")) {
            // Сгенерировать PDF-файл, используя указанные параметры
            viewer.view(viewOptions);
        }
    }
}
```

**Объяснение:**
- **Конфигурация пути:** Определите, где будут храниться визуализированные изображения.
- **PngViewOptions:** Укажите, что нам нужен вывод в формате PNG, и настройте форматирование пути для каждой страницы.
- **Размер исходной страницы визуализации:** Эта важная настройка гарантирует, что страницы не будут масштабироваться и сохранят свои исходные размеры.

#### Советы по устранению неполадок
Если у вас возникли проблемы:
- Обеспечить пути в `outputDirectory` и `"YOUR_DOCUMENT_DIRECTORY/SAMPLE_PDF"` верны.
- Проверьте правильность настройки GroupDocs.Viewer в вашем инструменте сборки.
