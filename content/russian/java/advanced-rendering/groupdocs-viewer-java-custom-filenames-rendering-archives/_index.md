
## Руководство по внедрению

Теперь давайте сосредоточимся на указании имен файлов при преобразовании архивов в PDF-файлы.

### Указание имени файла при рендеринге архивных файлов

Эта функция позволяет вам настраивать выходное имя файла вашего визуализированного PDF-документа. Вот как:

#### Шаг 1: Определите выходной каталог и путь к файлу

Начните с настройки выходного каталога и пути к файлу, используя заполнители для легкой настройки:

```java
import java.nio.file.Path;
// Определить выходной каталог и путь к файлу
Path outputDirectory = Path.of("YOUR_OUTPUT_DIRECTORY");
Path outputFilePath = outputDirectory.resolve("output.pdf");
```

#### Шаг 2: Инициализация объекта Viewer

Создать `Viewer` объект с архивным файлом, который вы хотите отрендерить:

```java
import com.groupdocs.viewer.Viewer;
try (Viewer viewer = new Viewer("YOUR_DOCUMENT_DIRECTORY/SAMPLE_ZIP")) {
    // Перейти к следующим шагам
} catch (Exception e) {
    e.printStackTrace();
}
```

#### Шаг 3: Создайте PdfViewOptions

Настраивать `PdfViewOptions` для указания конфигураций рендеринга:

```java
import com.groupdocs.viewer.options.PdfViewOptions;
// Настройте параметры просмотра PDF-файла
PdfViewOptions viewOptions = new PdfViewOptions(outputFilePath);
```

#### Шаг 4: Задайте пользовательское имя файла

Использовать `ArchiveOptions` чтобы задать пользовательское имя файла для вашего визуализированного PDF-документа:

```java
import com.groupdocs.viewer.options.FileName;
import com.groupdocs.viewer.options.ArchiveOptions;
// Укажите имя выходного PDF-файла
viewOptions.getArchiveOptions().setFileName(new FileName("my_custom_filename.pdf"));
```

#### Шаг 5: Преобразовать архивный файл в PDF

Наконец, выполните рендеринг вашего архивного файла, используя указанные параметры:

```java
// Выполнить процесс рендеринга
viewer.view(viewOptions);
```

### Советы по устранению неполадок
- Убедитесь, что все пути указаны правильно и каталоги существуют.
- Убедитесь, что у вас установлена правильная версия GroupDocs.Viewer.
