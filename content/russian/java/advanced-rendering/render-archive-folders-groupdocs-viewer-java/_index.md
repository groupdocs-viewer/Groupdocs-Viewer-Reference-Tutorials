## Настройка GroupDocs.Viewer для Java

### Конфигурация Maven

Чтобы интегрировать GroupDocs.Viewer в свой проект, добавьте следующие конфигурации в свой `pom.xml` файл:

```xml
<repositories>
   <repository>
      <id>groupdocs-repo</id>
      <name>GroupDocs Repository</name>
      <url>https://releases.groupdocs.com/viewer/java/</url>
   </repository>
</repositories>

<dependencies>
   <dependency>
      <groupId>com.groupdocs</groupId>
      <artifactId>groupdocs-viewer</artifactId>
      <version>25.2</version>
   </dependency>
</dependencies>
```

## Руководство по внедрению

В этом разделе мы рассмотрим, как визуализировать определенные папки в архиве с помощью GroupDocs.Viewer для Java.

### Функция: Рендеринг архивной папки

Эта функция позволяет выборочно визуализировать папку внутри архивного файла. Вот как:

#### Определить выходной путь

Настройте путь к выходному каталогу следующим способом:

```java
import java.nio.file.Path;
import java.nio.file.Paths;

public static Path definePath() {
    return Paths.get("YOUR_OUTPUT_DIRECTORY", "RenderArchiveFolder");
}
```
При таком подходе указывается, где будут храниться обработанные HTML-файлы.

#### Конкретная папка для рендеринга

Далее настройте параметры рендеринга и выполните:

```java
import com.groupdocs.viewer.Viewer;
import com.groupdocs.viewer.options.HtmlViewOptions;

public static void renderArchiveFolder() {
    Path outputDirectory = definePath();
    Path pageFilePathFormat = outputDirectory.resolve("page_{0}.html");

    HtmlViewOptions viewOptions = HtmlViewOptions.forEmbeddedResources(pageFilePathFormat);
    viewOptions.getArchiveOptions().setFolder("ThirdFolderWithItems");

    try (Viewer viewer = new Viewer("YOUR_DOCUMENT_DIRECTORY/SAMPLE_ZIP_WITH_FOLDERS")) {
        viewer.view(viewOptions);
    }
}
```

**Пояснение параметров:**
- `pageFilePathFormat`: Определяет шаблон именования для каждого вывода страницы.
- `viewOptions.getArchiveOptions().setFolder(...)`: Указывает, какую папку в архиве следует визуализировать.

### Функция: Определение пользовательского пути для выходного каталога

Для большей гибкости используйте служебную функцию, чтобы настроить выходной путь:

```java
public static Path definePath() {
    return Paths.get("YOUR_OUTPUT_DIRECTORY", "RenderArchiveFolder");
}
```
