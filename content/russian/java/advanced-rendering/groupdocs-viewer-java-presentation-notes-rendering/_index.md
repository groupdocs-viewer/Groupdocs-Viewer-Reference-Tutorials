## Понимание лицензирования GroupDocs Viewer для Java
Лицензирование GroupDocs Viewer определяет, какие функции будут доступны. Без действующей лицензии вывод может содержать водяные знаки или быть ограничен определённым количеством страниц. Обязательно загрузите файл лицензии перед рендерингом больших или коммерческих документов.

## Руководство по реализации

### Функция: Отображение презентации с заметками
В этом разделе показано, как отобразить файл PPTX в HTML, включая заметки докладчика.

#### Шаг 1: Определите каталог вывода и формат файла
Настройте папку, в которой будут сохраняться HTML‑страницы:

```java
import java.nio.file.Path;
import java.nio.file.Paths;

Path YOUR_DOCUMENT_DIRECTORY = Paths.get("YOUR_DOCUMENT_DIRECTORY");
Path pageFilePathFormat = YOUR_OUTPUT_DIRECTORY.resolve("page_{0}.html");
```

#### Шаг 2: Настройте параметры просмотра
Создайте параметры просмотра, которые встраивают ресурсы и включают рендеринг заметок:

```java
import com.groupdocs.viewer.options.HtmlViewOptions;

HtmlViewOptions viewOptions = HtmlViewOptions.forEmbeddedResources(pageFilePathFormat);
viewOptions.setRenderNotes(true); // Enable note rendering
```

> **Pro tip:** `forEmbeddedResources` создает автономный HTML, что упрощает развертывание на веб‑серверах.

#### Шаг 3: Загрузите и отобразите документ
Наконец, отобразите файл PPTX, используя параметры, определённые выше:

```java
try (Viewer viewer = new Viewer(YOUR_DOCUMENT_DIRECTORY.resolve("TestFiles.PPTX_WITH_NOTES"))) {
    // Render document to HTML with notes included
    viewer.view(viewOptions);
}
```

**Совет по устранению неполадок:** Убедитесь, что пути к файлам существуют и доступны для чтения. Отсутствующий файл вызовет `FileNotFoundException`.

## Java конвертация презентации в веб: внедрение результата
HTML‑файлы, сгенерированные приведённым выше кодом, могут быть обслужены напрямую из вашего веб‑приложения. Поскольку ресурсы встроены, достаточно скопировать папку вывода в каталог статического контента и сослаться на первый файл `page_0.html` в `<iframe>` или обычном `<div>`.

## Соображения по производительности
- Используйте **try‑with‑resources** для автоматического закрытия `Viewer` и освобождения памяти.  
- Кешируйте сгенерированный HTML для часто запрашиваемых презентаций, чтобы снизить нагрузку на CPU.  
- Следите за использованием кучи JVM при обработке больших PPTX‑файлов; при необходимости увеличьте размер кучи, если возникнет `OutOfMemoryError`.

## Распространённые проблемы и решения
| Проблема | Решение |
|----------|----------|
| **Заметки не отображаются** | Убедитесь, что перед рендерингом вызвано `viewOptions.setRenderNotes(true)`. |
| **Медленный рендеринг больших файлов** | Включите кеширование и рассматривайте возможность рендеринга страниц по запросу, а не сразу всех. |
| **Ошибки путей к файлам** | Используйте `Paths.get(...)` и дважды проверьте относительные и абсолютные пути. |
