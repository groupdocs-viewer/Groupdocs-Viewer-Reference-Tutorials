# Tutoriales de GroupDocs.Viewer

Bienvenido al centro de tutoriales de GroupDocs.Viewer. Aquí encontrarás una colección completa de tutoriales y guías para ayudarte a **add watermark document** y dominar nuestras potentes APIs de visor de documentos para .NET y Java. Ya sea que estés construyendo una aplicación web, una aplicación de escritorio o una solución móvil, GroupDocs.Viewer te permite renderizar y mostrar sin problemas una amplia variedad de formatos de documentos, incluidos PDF, Microsoft Office (Word, Excel, PowerPoint), CAD, imágenes y más.

## Respuestas rápidas
- **What does “add watermark document” mean?** Se refiere a incrustar una superposición de texto o imagen semitransparente en cada página de un documento renderizado.  
- **Which formats support watermarks?** Todos los formatos renderizados por GroupDocs.Viewer, incluidos PDF, DOCX, XLSX, CAD y mensajes de correo electrónico.  
- **Do I need a license?** Se requiere una licencia válida de GroupDocs.Viewer para uso en producción; hay una prueba gratuita disponible.  
- **Can I combine watermarks with other features?** Sí—las marcas de agua pueden aplicarse junto con el almacenamiento en caché, renderizado personalizado y configuraciones de seguridad.  
- **Is it compatible with .NET Core?** Absolutamente—GroupDocs.Viewer funciona con .NET Framework, .NET Core y .NET 5/6+.

## Cómo agregar Watermark Document con GroupDocs.Viewer

Agregar una marca de agua a menudo es necesario para la marca, la confidencialidad o el cumplimiento normativo. Con GroupDocs.Viewer puedes aplicar una marca de agua mientras renderizas un documento a formatos HTML, PDF o de imagen. El proceso es sencillo:

1. **Crear un objeto `WatermarkOptions`** y establecer el texto, color, opacidad y rotación.  
2. **Pasar las opciones** al método de renderizado apropiado (p. ej., `RenderToPdf`, `RenderToHtml` o `RenderToImage`).  
3. **Renderizar el documento**—la salida contendrá la marca de agua en cada página.

Este enfoque funciona en todos los tipos de archivo compatibles, incluidos los escenarios **render pdf .net**, dibujos CAD y mensajes de correo electrónico.

## Preguntas frecuentes

**Q: ¿Puedo agregar una marca de agua a documentos renderizados como imágenes?**  
A: Sí—utiliza `WatermarkOptions` junto con `RenderToImage` para incrustar marcas de agua en cada página de imagen generada.

**Q: ¿Afecta la adición de una marca de agua al rendimiento del renderizado?**  
A: El impacto es mínimo; GroupDocs.Viewer optimiza el proceso de superposición, especialmente cuando se combina con caché.

**Q: ¿Se admiten marcas de agua al renderizar mensajes de correo electrónico?**  
A: Absolutamente—las marcas de agua pueden aplicarse a la salida HTML o PDF de los mensajes de correo electrónico renderizados.

**Q: ¿Cómo personalizo la apariencia de la marca de agua?**  
A: Puedes establecer la familia de fuentes, tamaño, color, opacidad, ángulo de rotación e incluso usar una imagen como marca de agua.

**Q: ¿Es posible aplicar diferentes marcas de agua a distintas páginas?**  
A: La API estándar aplica una marca de agua uniforme, pero puedes implementar lógica de renderizado personalizada para variar las marcas de agua por página.
