# Tutoriels GroupDocs.Viewer

Bienvenue sur le hub de tutoriels GroupDocs.Viewer. Vous y trouverez une collection complète de tutoriels et de guides pour vous aider à **add watermark document** et à maîtriser nos puissantes API de visualisation de documents pour .NET et Java. Que vous développiez une application web, une application de bureau ou une solution mobile, GroupDocs.Viewer vous permet de rendre et d’afficher de manière transparente une grande variété de formats de documents, y compris PDF, Microsoft Office (Word, Excel, PowerPoint), CAD, images, et plus encore.

## Réponses rapides
- **Que signifie “add watermark document” ?** Il s’agit d’insérer une superposition de texte ou d’image semi‑transparente sur chaque page d’un document rendu.  
- **Quels formats prennent en charge les filigranes ?** Tous les formats rendus par GroupDocs.Viewer, y compris PDF, DOCX, XLSX, CAD et les messages électroniques.  
- **Ai‑je besoin d’une licence ?** Une licence valide de GroupDocs.Viewer est requise pour une utilisation en production ; un essai gratuit est disponible.  
- **Puis‑je combiner les filigranes avec d’autres fonctionnalités ?** Oui — les filigranes peuvent être appliqués conjointement avec la mise en cache, le rendu personnalisé et les paramètres de sécurité.  
- **Est‑il compatible avec .NET Core ?** Absolument — GroupDocs.Viewer fonctionne avec .NET Framework, .NET Core et .NET 5/6+.

## Comment ajouter un filigrane de document avec GroupDocs.Viewer

L’ajout d’un filigrane est souvent nécessaire pour le branding, la confidentialité ou la conformité réglementaire. Avec GroupDocs.Viewer, vous pouvez appliquer un filigrane lors du rendu d’un document en HTML, PDF ou formats d’image. Le processus est simple :

1. **Créer un `WatermarkOptions` object** and set the text, color, opacity, and rotation.  
2. **Pass the options** to the appropriate rendering method (e.g., `RenderToPdf`, `RenderToHtml`, or `RenderToImage`).  
3. **Render the document**—the output will contain the watermark on every page.

Cette approche fonctionne avec tous les types de fichiers pris en charge, y compris les scénarios **render pdf .net**, les dessins CAD et les messages électroniques.

## Questions fréquentes

**Q : Puis‑je ajouter un filigrane aux documents rendus sous forme d’images ?**  
R : Oui — utilisez `WatermarkOptions` avec `RenderToImage` pour intégrer des filigranes sur chaque page d’image générée.

**Q : L’ajout d’un filigrane affecte‑t‑il les performances de rendu ?**  
R : L’impact est minime ; GroupDocs.Viewer optimise le processus de superposition, surtout lorsqu’il est combiné avec la mise en cache.

**Q : Les filigranes sont‑ils pris en charge lors du rendu de messages électroniques ?**  
R : Absolument — les filigranes peuvent être appliqués à la sortie HTML ou PDF des messages électroniques rendus.

**Q : Comment personnaliser l’apparence du filigrane ?**  
R : Vous pouvez définir la famille de police, la taille, la couleur, l’opacité, l’angle de rotation, et même utiliser une image comme filigrane.

**Q : Est‑il possible d’appliquer différents filigranes à différentes pages ?**  
R : L’API standard applique un filigrane uniforme, mais vous pouvez implémenter une logique de rendu personnalisée pour varier les filigranes page par page.
