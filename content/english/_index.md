## How to add watermark to document with GroupDocs.Viewer
Adding a watermark is often the final step in preparing a document for distribution. With GroupDocs.Viewer you can apply text or image watermarks to any supported format before rendering it to HTML, JPEG, PNG, or PDF. This ensures that proprietary content remains identifiable and protected throughout its lifecycle.

### Why add watermark to document?
- **Brand protection:** Reinforce company branding on every page.  
- **Security:** Deter unauthorized copying or redistribution.  
- **Compliance:** Meet legal or regulatory requirements for document marking.

### Simple steps to add a watermark
1. **Load the document** using the Viewer API.  
2. **Configure the watermark settings** (text, opacity, position).  
3. **Render the document** – the watermark is applied automatically during rendering.

You’ll find detailed code samples in the dedicated .NET and Java watermark tutorials linked below.

## How to render PDF in .NET with GroupDocs.Viewer
If you’re wondering **how to render pdf .net**, GroupDocs.Viewer makes it effortless. The library handles complex PDFs, preserves layout, and supports high‑resolution image output. This capability is essential for building preview features, generating thumbnails, or converting PDFs to web‑friendly formats.

### Benefits of rendering PDF in .NET
- **Accurate layout preservation** – pages look exactly as the original.  
- **Performance‑optimized** – stream rendering for large files.  
- **Cross‑platform** – works with .NET Framework, .NET Core, and .NET 5/6.

## Frequently Asked Questions

**Q: Can I add both text and image watermarks to the same document?**  
A: Yes. GroupDocs.Viewer lets you stack multiple watermarks and control their order, opacity, and positioning.

**Q: Is there a limit to the number of pages I can render at once?**  
A: The library streams pages, so you can render large documents efficiently. You can also render specific page ranges to conserve resources.

**Q: Does the “how to render pdf .net” feature support password‑protected PDFs?**  
A: Absolutely. You can pass the password when loading the document, and rendering proceeds normally.

**Q: Are the watermark settings configurable per page?**  
A: Yes. You can apply different watermarks to individual pages or to the entire document.

**Q: What platforms are supported for cloud rendering?**  
A: The API works with any platform that can make HTTP calls – including Azure Functions, AWS Lambda, and on‑premises servers.
